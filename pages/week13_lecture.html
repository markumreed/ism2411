Today's walkthrough

Why pandas instead of csv + lists

In Unit 3, you used the csv module to read files row by row into lists of strings. That works — but consider what it takes to find the top 10 rows by revenue: you'd write a loop, convert the revenue column from string to float, sort a list of dictionaries, and slice the result. That's 15–20 lines of code, and every step is a potential bug.

pandas does the same thing in one chained expression: df.sort_values("revenue", ascending=False).head(10). Beyond brevity, pandas is dramatically faster because its core operations run in compiled C code under the hood. On a 500,000-row file, a pandas sort completes in milliseconds. A Python list sort on the same data can take seconds.

pandas also has a decisive advantage over Excel for business work: it handles datasets of any size that fits in memory, it produces identical results every run (no hidden formulas, no accidentally overwritten cells), and every step is recorded in code that can be reviewed, version-controlled, and reused. A pandas script is a permanent, auditable record of your analysis.

Loading a CSV

import pandas as pd

df = pd.read_csv("sales.csv")
df.head()        # first 5 rows
df.tail(10)      # last 10 rows
df.shape         # (rows, columns)
df.columns       # column names
df.dtypes        # data types per column
df.info()        # full summary: types + non-null counts

A DataFrame is a table

Each column is a Series — a labeled array of uniform data. The whole table is a DataFrame. You can think of it as a dictionary where each key is a column name and each value is a list (but with a shared index tying all columns together). This mental model explains almost every pandas behavior you'll encounter.

Creating a small DataFrame by hand is a great way to test ideas before running them on real data. The constructor accepts a dictionary of lists, which maps directly onto the dictionary mental model.

df = pd.DataFrame({
    "item":  ["pen", "book", "tape"],
    "price": [1.50, 12.00, 3.25],
    "stock": [120, 8, 47],
})

print(df)
print(df["price"])           # a single column (Series)
print(df["price"].mean())    # 5.583...
print(df[["item","price"]])  # two columns (DataFrame)

Filtering rows

Filtering in pandas uses "boolean indexing." You create a condition that evaluates to True or False for each row, then pass that condition inside brackets. pandas returns only the rows where the condition is True. This mirrors the logic of an Excel AutoFilter but is reusable, composable, and works on any dataset size.

The critical syntax rule: use & for AND and | for OR. Wrap each condition in its own parentheses when combining. Python's and/or keywords do not work with pandas boolean arrays and will raise a ValueError.

south = df[df["region"] == "South"]
big = df[df["revenue"] > 1000]
combo = df[(df["region"] == "South") & (df["revenue"] > 1000)]
# note: & not and, and parens around each condition

# Negate a condition with ~
not_south = df[~(df["region"] == "South")]

Sorting

Sorting by a column and taking the top or bottom N rows is a pattern you will use constantly — for leaderboards, exception reports, and identifying outliers. The result of sort_values() is always a new DataFrame unless you pass inplace=True.

df.sort_values("revenue", ascending=False).head(10)

# Sort by multiple columns (region, then revenue within region)
df.sort_values(["region", "revenue"], ascending=[True, False])

Live Coding Walkthrough

This is the complete script we build together during class. Run it cell by cell in Jupyter or as a single Python file.

import pandas as pd

# ── Step 1: Load ──────────────────────────────────────────────
df = pd.read_csv("data/retail_sales.csv")
print("Shape:", df.shape)
print(df.head())
print(df.dtypes)

# ── Step 2: Explore ───────────────────────────────────────────
print("\nColumns:", df.columns.tolist())
print("\nRegions:", df["region"].unique())
print("\nMissing values:")
print(df.isnull().sum())

# ── Step 3: Filter ────────────────────────────────────────────
south = df[df["region"] == "South"]
print(f"\nSouth region: {len(south)} rows")

big_south = df[(df["region"] == "South") & (df["revenue"] > 1000)]
print(f"Big South orders (>$1000): {len(big_south)} rows")

# ── Step 4: Top 10 by revenue ─────────────────────────────────
top10 = df.sort_values("revenue", ascending=False).head(10)
print("\nTop 10 orders by revenue:")
print(top10[["region", "product", "revenue"]])

# ── Step 5: Add a calculated column ──────────────────────────
df["per_unit"] = df["revenue"] / df["quantity"]
print("\nWith per_unit column:")
print(df.head())

# ── Step 6: Save ──────────────────────────────────────────────
df.to_csv("data/explored.csv", index=False)
print("\nSaved to data/explored.csv")

Common Errors

Error 1: Using and in a filter

# WRONG — raises ValueError
df[df["region"] == "South" and df["revenue"] > 1000]

# RIGHT — use & with parentheses
df[(df["region"] == "South") & (df["revenue"] > 1000)]

Error 2: FileNotFoundError when loading CSV

# Error: FileNotFoundError: [Errno 2] No such file or directory: 'retail_sales.csv'
df = pd.read_csv("retail_sales.csv")

# Fix: use the correct relative path from your script's location
df = pd.read_csv("data/retail_sales.csv")
# Or use an absolute path for debugging:
import os
print(os.getcwd())  # see where Python thinks you are

Error 3: AttributeError after sorting

# WRONG — sort_values returns a new DataFrame; you can't chain without assigning
df.sort_values("revenue", ascending=False)
print(df.head())  # df is still unsorted!

# RIGHT — assign the result
df_sorted = df.sort_values("revenue", ascending=False)
print(df_sorted.head())

Key Takeaways
  • pandas replaces csv + list code with fast, readable, composable operations on DataFrames.
  • A DataFrame is a table; each column is a Series; use & and | (never and/or) for multi-condition filters.
  • Most pandas methods return new objects — assign the result or use inplace=True.
  • The workflow — load, inspect, filter, sort, add columns, save — is the same skeleton you will reuse in every lab and the capstone.
  • pandas scales to millions of rows; it is reproducible, version-controllable, and audit-friendly in ways Excel is not.
