Why this matters

Real business data does not live inside your script. It lives in CSV files on shared drives, in reports exported from accounting software, in data dumps from e-commerce platforms and point-of-sale systems. Every week, analysts at thousands of companies manually open these files in Excel, clean them, and paste results into other spreadsheets. Learning to do this in Python means you can automate the entire pipeline — and run it again tomorrow with zero additional effort.

Consider a retail operations analyst who receives a daily sales export from the POS system: 500 rows, each with a product name, quantity, and unit price. In Excel, she spends 20 minutes adding a "Total" column, filtering for high-value orders, and copying the results to a summary sheet. In Python, that same workflow takes 15 lines of code and runs in under a second. Next Monday she runs the same script on the new file. The week after that she does the same thing. She just got back 17 hours a month — and the analysis is reproducible and auditable.

File I/O is also the gateway to the rest of the data stack. pandas, the library you'll use in Unit 4, reads CSV files with a single function call. But understanding what happens underneath — how files are opened, how rows are parsed, why the with statement matters — makes you a far better pandas user than someone who skipped straight to pd.read_csv() without knowing what it's actually doing. Core Concepts

open() and modes

open(filename, mode) — modes: "r" (read), "w" (write, overwrites everything), "a" (append to end), "r+" (read and write). Always specify the mode explicitly.

with statement

with open("file.txt", "r") as f: automatically closes the file when the block ends — even if an error occurs inside. This is the safest and most Pythonic way to work with files.

csv.reader

Reads a CSV file row by row, returning each row as a list of strings. Handles commas inside quoted fields correctly. Use next(reader) to skip the header row.

csv.writer

Writes rows to a CSV. writer.writerow([...]) for one row; writer.writerows([...]) for multiple. Always pass newline="" to open() on Windows to avoid extra blank lines.

Relative paths

"data/sales.csv" means the data/ folder inside wherever your script runs. Relative paths work on any machine. Absolute paths like /Users/alice/... break when anyone else clones your repo.

Business Scenario

Your manager sends you sales.csv — a weekly export from the point-of-sale system. Each row has a product name, unit price, and quantity sold. You need to compute the revenue for each row, filter to orders over $100, and write the results to cleaned.csv for the finance team. Here is the full Python workflow:

import csv

# --- Step 1: Read and process ---
processed = []

with open("data/sales.csv", "r") as f:
    reader = csv.reader(f)
    header = next(reader)   # skip the header row

    for row in reader:
        product, price_str, qty_str = row
        price = float(price_str)
        qty   = int(qty_str)
        total = price * qty
        processed.append({"product": product, "price": price,
                           "qty": qty, "total": total})

# --- Step 2: Filter high-value orders ---
high_value = [r for r in processed if r["total"] > 100]

# --- Step 3: Write cleaned output ---
with open("data/cleaned.csv", "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerow(["Product", "Price", "Qty", "Total"])
    for r in high_value:
        writer.writerow([r["product"], r["price"], r["qty"], f"{r['total']:.2f}"])

print(f"Wrote {len(high_value)} high-value orders to cleaned.csv")

Key Vocabulary

  • File mode — the string passed as the second argument to open(): "r" read, "w" write (overwrites), "a" append, "r+" read and write
  • with statement — a context manager that automatically closes the file when the indented block exits, even if an exception occurs
  • csv module — Python's built-in module for reading and writing CSV files; handles quoted fields, escaped commas, and platform line endings
  • csv.reader — returns an iterator over the rows of a CSV file; each row is a list of strings
  • csv.DictReader — like csv.reader, but returns each row as a dictionary using the first row as keys; more readable for files with many columns
  • next(reader) — advances the iterator by one row; used to skip the header row before the loop
  • Relative path — a file path specified relative to the current working directory; portable across machines
  • Absolute path — a file path from the root of the filesystem; breaks when the project moves to a different machine
Common Misconceptions

Forgetting to close files: If you open a file without the with statement and forget to call f.close(), the file may not be fully written to disk, and on some systems the data will be lost. Always use with open(...) as f:.

Hardcoded absolute paths: open("/Users/alice/Documents/ism2411/data/sales.csv") works on your machine and nowhere else. Anyone who clones your repo gets a FileNotFoundError. Always use relative paths: open("data/sales.csv").

"w" mode silently deletes data: Opening a file with mode "w" immediately truncates (empties) the file, even before you write anything. If you meant to append, use "a". Forgetting this has cost analysts hours of work.

Q2 · Code Trace
What does this code do?
import csv
with open("sales.csv", "r") as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)
Answer
Opens sales.csv and prints each row as a list of strings (including the header row)

What is the main advantage of using the with statement when opening a file?

  • b) It automatically closes the file when the block exits, even if an error occurs

Q7 · Code Trace
What does next(reader) do when called before the for row in reader loop?
Answer
It advances the reader by one row and returns that row (usually the header). The subsequent for loop then starts from the second row, skipping the header.
Q8 · Multiple Choice
What is the difference between csv.reader and csv.DictReader?
  • a) DictReader is slower
  • b) reader returns each row as a list; DictReader returns each row as a dictionary with the header as keys
  • c) DictReader can only read files with fewer than 10 columns
  • d) There is no difference
Answer
b) reader returns each row as a list; DictReader returns each row as a dictionary using the header row as keys — making column access by name possible: row["price"] instead of row[1]
Q9 · Bug Fix
This code is supposed to compute revenue from each row. What is the bug?
import csv
with open("sales.csv", "r") as f:
    reader = csv.reader(f)
    for row in reader:
        product, price, qty = row
        revenue = price * qty
        print(product, revenue)
Answer
csv.reader returns all values as strings. Multiplying two strings produces string repetition, not arithmetic. Fix: convert types before computing: revenue = float(price) * int(qty).
