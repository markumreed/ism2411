Open Extensions

In VS Code, go to File → New File. Save it as hello.py on your Desktop (use ⌘S on Mac or Ctrl+S on Windows).

Troubleshooting — Common Problems and Fixes
⚠ Problem
python is not recognized on Windows
✓ Fix
You missed the “Add Python to PATH” checkbox during installation. Go to Settings → Apps, uninstall Python, then reinstall with that box checked.
⚠ Problem
command not found: python on Mac
✓ Fix
On Mac the command is python3, not python. Both work in VS Code once the extension is installed. Always use python3 in the terminal.
⚠ Problem
VS Code says “No Python interpreter selected”
✓ Fix
Press Ctrl+Shift+P (or ⌘Shift+P on Mac). Type Python: Select Interpreter and press Enter. Choose the Python 3.12 entry from the list.
⚠ Problem
Multiple Python versions are installed
✓ Fix
Common on Mac. As long as the VS Code interpreter is set to Python 3.10 or higher, you are fine. You can check the interpreter version in the bottom-right corner of VS Code when a .py file is open.
⚠ Problem
Right-click menu does not show “Run Python File in Terminal”
✓ Fix
The Python extension is not installed or not enabled. Open Extensions (Ctrl+Shift+X), confirm the Microsoft Python extension shows Disable (not Install). Reload VS Code after installation.
⚠ Problem
The terminal runs but prints an error instead of the output
✓ Fix
Read the error message carefully. A SyntaxError usually means a typo in the code — check your quotation marks and spelling. A ModuleNotFoundError means a missing package (not expected at this stage).
Getting Help Before Day 1
First: two minutes with the duck

Before posting anywhere, try the oldest trick in software: explain the problem out loud to an inanimate object — a rubber duck, a mug, anything patient. This is rubber duck debugging, from The Pragmatic Programmer (Hunt & Thomas, 1999), and it works on setup problems just as well as code:

1. Say what the step is supposed to do. 2. Say what you actually typed and what actually happened — literally, no summarizing. 3. The moment those two don’t match, you’ve found the problem.

Out loud it sounds like: “Step 4 says run source .venv/bin/activate from the project folder. I’m running it… from my home folder. That’s a different folder.” Most setup errors die right there. And if yours doesn’t, what you just said out loud is exactly what your help post should say. You’ll meet the duck again properly in Module 7, where debugging is taught for real.

Post in the course discussion board with your operating system and version, which step you are on, and a screenshot of the error message. The instructor monitors the board before the semester starts. The first 15 minutes of Week 1 are also reserved for setup troubleshooting — bring your laptop.
