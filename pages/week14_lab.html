Clean a Messy Sales CSV

Apply the full cleaning workflow to a deliberately messy version of the retail dataset. Every decision you make here will feed directly into Module 15's analysis and the capstone.

Exercises

EXERCISE 1
BASIC
Inspect
Load messy_sales.csv. Run .info() and .isnull().sum(). In a comment block, write down at least 3 data quality issues you observe. Be specific: "column X has Y% missing" or "column Z is dtype object but should be float."
Expected output: .info() shows mismatched dtypes and non-null counts below total; .isnull().sum() shows counts per column.
EXERCISE 2
BASIC
Standardize column names
In one line, convert all column names to lowercase and replace spaces with underscores. Print the updated column list to confirm.
Expected output: a list like ['order_date', 'product_category', 'region', 'unit_price', 'quantity_sold', 'customer_id']
EXERCISE 3
INTERMEDIATE
Handle missing values
For numeric columns with few missing values, fill with the column mean. For rows missing essential identifiers (order_id, order_date), drop them. Add a comment explaining each choice. Print the shape before and after to confirm rows were dropped.
Expected output: shape shrinks after dropping essential-column rows; isnull().sum() drops to 0 for targeted columns.
EXERCISE 4
INTERMEDIATE
Fix types
Convert order_date to datetime with pd.to_datetime(..., errors="coerce"). Convert unit_price (strip "$" if needed) and quantity_sold to numeric types with .astype(). Print .dtypes to confirm the conversion.
Expected output: dtypes shows datetime64 for order_date, float64 for unit_price, int64 for quantity_sold.
EXERCISE 5
INTERMEDIATE
Check duplicates
Run df.duplicated().sum() to count duplicate rows. If duplicates exist, remove them with df.drop_duplicates(). Print the shape again to confirm.
EXERCISE 6
INTERMEDIATE
Describe
Run .describe() on the cleaned data. In a comment, identify at least two statistics that look suspicious — a negative revenue, an absurdly high max, a mean and median far apart. Propose a business explanation for each.
Expected output: describe() table; your comments demonstrate analytical thinking, not just code execution.
EXERCISE 7
BASIC
Save
Export the cleaned data to data/clean_sales.csv with index=False. Then reload it with pd.read_csv() and print .info() to verify the types were preserved correctly.

Stretch Challenge

STRETCH 1 · CHALLENGE
Detect and flag outliers
Use the IQR method to flag outliers in the unit_price column. Compute Q1, Q3, and IQR. Create a new column is_outlier that is True when a price is more than 1.5 × IQR above Q3 or below Q1. Print how many outliers there are and their average price.
STRETCH 2 · CHALLENGE
Write a reusable cleaning function
Wrap your entire cleaning pipeline (steps 2–7) in a function called clean_sales(filepath) that takes a file path and returns a clean DataFrame. Call the function and print the result's shape and .dtypes to verify it works.

Reflection

Self-assessment

1. Which of the three cleaning decisions — drop, fill, or flag — feels most ambiguous to you? What additional information would you want before making that call on real data?

2. Look at your comments. If someone who didn't know the dataset read your script, would they understand why you made each decision? Revise any comment that only says what you did without saying why.

3. After running describe(), what is the most important thing you noticed about this dataset that you did not expect when you first loaded it?
