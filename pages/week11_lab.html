Customer Dictionary & Lookup

Build customer records as dictionaries, use tier-based lookups, and process a list of customers with a loop. This is the pattern that underlies every real CRM and analytics workflow.

Module 11 · Lab Unit 3 GitHub submission

Exercises

EXERCISE 1 · Beginner
One customer
Create one customer dict with at least 4 fields: name, email, tier, ytd_spend. Print each field individually using key access. Then print the whole dict.
Expected: each field on its own line, then the full dict
EXERCISE 2 · Beginner
Update and add
Add a phone field. Update the tier to "platinum". Delete the email field with del. Print the dict after each step so you can see it change.
Expected: 3 printed versions of the dict showing each change
EXERCISE 3 · Beginner
Safe lookup
Use .get() to look up the email field you deleted in Exercise 2 — provide the default "no email on file". Confirm it returns the default rather than crashing. Then use bracket notation to look up the same key and observe the KeyError.
Expected: .get() returns default; bracket notation raises KeyError (catch it with try/except)
EXERCISE 4 · Beginner
Iterate
Print every key:value pair using a for loop with .items(), formatted as "name: Alice". Then try iterating without .items() and explain in a comment what the difference is.
Expected: each field formatted on its own line
EXERCISE 5 · Intermediate
Lookup table
Build tier_discounts = {"bronze": 0, "silver": .05, "gold": .10, "platinum": .15}. Look up the discount for the customer's current tier using .get() with default 0. Compute the discounted price for a $200 purchase. Print the tier, discount rate, and final price.
Expected: tier · discount % · final price all printed
EXERCISE 6 · Intermediate
List of dicts
Build a list customers = [dict1, dict2, dict3] with at least 3 different customers (different tiers). Loop through it and for each customer print their name, tier, and the discount they receive on a $300 purchase using the tier_discounts lookup table.
Expected: one formatted line per customer showing name, tier, discount amount
EXERCISE 7 · Intermediate
Nested catalog
Build a nested product catalog: catalog = {"Widget A": {"price": 9.99, "stock": 150}, "Gadget C": {"price": 49.99, "stock": 8}}. Loop through the catalog and print any product with stock below 20 as a "LOW STOCK" alert.
Expected: only Gadget C triggers the alert
EXERCISE 8 · Advanced
Sales by region
Given a list of transaction dictionaries (each has region and amount), build a summary dictionary where each key is a region name and the value is the total sales for that region. Use a loop and .get(region, 0) to accumulate. Print the final summary sorted by region name.
Sample input: [{"region": "South", "amount": 300}, {"region": "North", "amount": 150}, {"region": "South", "amount": 200}] → South: 500, North: 150

Stretch Challenge

Write a function customer_report(customers, tier_discounts, purchase) that accepts a list of customer dictionaries, a tier-to-discount mapping, and a purchase amount. It should print a formatted table with columns: Name, Tier, Discount, Final Price. Call it with at least 4 customers spanning all four tiers.

Reflection

Before you submit

In a Python comment at the top of your file, answer these two questions in 2-3 sentences each:

  1. In Exercise 3 you saw the difference between .get() and bracket notation. When in real work would you use each one, and what's your rule of thumb for choosing?
  2. Exercise 8 asked you to build a summary dictionary with a loop. What does this pattern remind you of from Excel, and why might the Python version be more useful for large datasets?
