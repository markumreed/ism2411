Lists & Tuples

Lists are the first collection. Anything that comes in groups — products, customers, sales, dates — lives in a list. After today, you will use lists every single module.

Today's walkthrough

Creating and indexing

A list is items in square brackets, separated by commas. Python creates the list and stores each item in a numbered slot called an index. The critical thing to remember: index counting starts at zero. If you have five products, their indices are 0, 1, 2, 3, 4 — not 1, 2, 3, 4, 5.

Negative indices count backwards from the end. inventory[-1] is always the last item regardless of how long the list is. This is extremely useful when you don't know the length in advance — for example, the most recently added item to a sales log.

You can also check membership with the in operator: "pen" in inventory returns True if "pen" is anywhere in the list. This is much cleaner than writing a loop to search manually.

inventory = ["pen", "notebook", "stapler", "tape"]

print(inventory[0])    # pen
print(inventory[2])    # stapler
print(inventory[-1])   # tape  (negative counts from the end)

# Membership test
print("pen" in inventory)   # True
print("ruler" in inventory) # False

Slicing

Get a sub-list with [start:stop]. The start index is included; the stop index is excluded. This "half-open interval" convention is consistent throughout Python — once you internalize it, slicing becomes second nature.

Omitting start means "from the beginning." Omitting stop means "to the end." You can combine these with negative indices. Slicing never modifies the original list — it always returns a brand new list.

nums = [10, 20, 30, 40, 50]
print(nums[1:4])   # [20, 30, 40]
print(nums[:3])    # [10, 20, 30]
print(nums[-2:])   # [40, 50]
print(nums[:])     # [10, 20, 30, 40, 50]  — a copy of the whole list

Common list methods

List methods modify the list in place and (mostly) return None. This catches many beginners: if you write sorted_items = items.sort(), you'll find sorted_items is None — because .sort() sorted the original and returned nothing. To get a sorted copy without changing the original, use the built-in sorted(items) function instead.

.pop() is particularly useful in business logic — it removes and returns the last item, so you can both retrieve and delete in one step. .remove() deletes the first occurrence of a value by matching the value itself (not an index). del items[i] deletes by index.

items = ["a", "b", "c"]

items.append("d")     # ["a", "b", "c", "d"]
items.remove("b")     # ["a", "c", "d"]
items.sort()          # sorts in place; returns None
last = items.pop()    # removes and returns "d"; items is now ["a", "c"]
items.reverse()       # reverses in place
print(len(items))     # 2

Tuples — when immutability matters

A tuple looks like a list with parentheses instead of brackets — but it cannot be changed after creation. Use tuples for fixed records like GPS coordinates, RGB colors, or database rows. The immutability is a feature, not a limitation: it signals to anyone reading your code that this data should not change, and Python will raise a TypeError if you try.

Tuples are also slightly faster than lists for reading because Python doesn't have to allocate extra memory for potential growth. For large numbers of fixed records — like millions of coordinate pairs — this matters.

You can unpack tuple values directly into variables, which is a clean way to work with fixed records:

coords = (40.7128, -74.0060)   # NYC lat/lon
color = (255, 128, 0)           # RGB orange

# Unpacking
lat, lon = coords
print(f"Latitude: {lat}, Longitude: {lon}")

# This would raise TypeError:
# coords[0] = 99   # TypeError: 'tuple' object does not support item assignment

When to use which

Use a list when the collection will change over time: an inventory you add to and remove from, a shopping cart, a daily sales log where you accumulate results with .append().

Use a tuple when the data is a fixed record that should not change: a GPS coordinate pair, an RGB color value, a database row you're reading, a function returning multiple values. The tuple communicates intent — this is a fixed unit of data, not a bag you keep stuffing things into.

Code Walkthrough

Accumulating results with a list

A very common pattern in business scripts is to start with an empty list and .append() to it as you process data. This is the Python equivalent of building a spreadsheet column row by row.

# Calculate revenue for each product and collect results
products = [("Widget A", 9.99, 50), ("Widget B", 14.99, 30), ("Gadget C", 24.99, 20)]
revenues = []

for name, price, qty in products:
    revenue = price * qty
    revenues.append(revenue)
    print(f"{name}: ${revenue:.2f}")

print(f"\nTotal revenue: ${sum(revenues):.2f}")
print(f"Average revenue: ${sum(revenues)/len(revenues):.2f}")

# Widget A: $499.50
# Widget B: $449.70
# Gadget C: $499.80
# Total revenue: $1449.00
# Average revenue: $483.00

Sorting a list of business data

You can sort a list of numbers or strings with .sort(). For descending order, pass reverse=True. The sorted() function does the same but returns a new list, leaving the original unchanged.

daily_sales = [340, 127, 589, 204, 467, 88, 731]

# Sort ascending (cheapest to most expensive day)
daily_sales.sort()
print(daily_sales)  # [88, 127, 204, 340, 467, 589, 731]

# Best day's sales
print("Best day:", daily_sales[-1])   # 731

# Worst day's sales
print("Worst day:", daily_sales[0])   # 88

Common Errors

IndexError: list index out of range

This happens when you try to access an index that doesn't exist. A list of 5 items has valid indices 0–4. Asking for index 5 causes an IndexError. This commonly appears when students try to access "the last item" by using the length as an index.

# WRONG — causes IndexError
items = ["a", "b", "c"]
print(items[3])   # IndexError: list index out of range

# RIGHT — use -1 for last item, or len(items)-1
print(items[-1])       # c
print(items[len(items)-1])  # c  (equivalent, but -1 is cleaner)

Modifying a list while iterating over it

Never add to or remove from a list inside a for loop that's iterating over that same list. Python's behavior in this case is unpredictable — items get skipped or processed twice. Instead, build a new list or collect the items to remove and process them after the loop.

# WRONG — unpredictable behavior
prices = [10, 20, 30, 40, 50]
for p in prices:
    if p < 25:
        prices.remove(p)  # modifying while iterating!

# RIGHT — filter to a new list
prices = [10, 20, 30, 40, 50]
prices = [p for p in prices if p >= 25]
print(prices)  # [30, 40, 50]

Key Takeaways
  • Lists are zero-indexed — items[0] is the first item, items[-1] is the last.
  • .sort() modifies in place and returns None; use sorted() for a new sorted list.
  • Use .append() to accumulate results in a loop — this is one of the most common Python patterns.
  • Tuples are immutable — use them for fixed records where accidental mutation would be a bug.
  • Never modify a list while iterating over it; build a new list instead.
