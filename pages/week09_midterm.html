MODULE 9 · MIDTERM EXAMINATION

Midterm Exam

A 75-minute in-class exam covering Units 1 and 2 (Modules 1–8). Closed-resource: no laptops, no phones, no AI tools. A single printed cheatsheet is allowed.

What this page tells you: exactly what's on the exam, exactly what isn't, what materials you may bring, and how to prepare.

Format

  • Date: Module 9, regular class meeting
  • Duration: 75 minutes
  • Format: Mix of multiple choice, short answer, and "what does this code print" / "fix this bug" problems
  • Materials allowed: One double-sided 8.5×11 sheet, handwritten or printed, prepared by you. Pen or pencil.
  • Materials NOT allowed: Laptop, phone, smartwatch, textbook, prior labs, AI tools of any kind
  • Weight: 20% of final grade

What's on it

The exam is built from ★ MIDTERM-ELIGIBLE quiz questions across Modules 1–8. Some questions appear verbatim from those quizzes. Others are variations — same concept, different numbers or context. Study the module quizzes and you will recognize what's on the exam.

Topics covered

  1. Module 1 — What is a Computer · hardware vs software, what code is, what an interpreter does
  2. Module 2 — Command Line · navigating directories, running scripts, common commands
  3. Module 3 — Variables & Data Types · int, float, str, bool, type conversion, variable naming
  4. Module 4 — Operators · arithmetic, comparison, logical, operator precedence
  5. Module 5 — Conditionals · if/elif/else, boolean logic, truthy/falsy values
  6. Module 6 — Loops · for, while, range(), break, continue
  7. Module 7 — Functions & Debugging · def, return, parameters, scope, reading errors
  8. Module 8 — Git & GitHub · workflow basics, commit/push/pull, why version control matters

What's NOT on it

  • pandas, DataFrames, plotting (Unit 4 — not yet covered)
  • Dictionaries or list comprehensions (Unit 3 — not yet covered)
  • File I/O beyond conceptual awareness
  • Anything from a Module 9+ topic

How to prepare

  1. Re-do every ★ Midterm-Eligible quiz. They're at the bottom of each Module 1–8 reading page. Aim for 100% without looking.
  2. Re-run your lab code. If your Module 5 lab fails because you forgot how if/elif chains work, find out before the exam.
  3. Print the cheatsheets for Unit 1 and Unit 2. Read them. Then study the cheatsheets for Units 1 and 2 side by side — note which concepts appear in both, and which are unique to each unit.
  4. Write 5 functions from scratch — no looking. Try calculate_tax(price, rate), apply_discount(price, pct), is_positive(n), count_above(nums, threshold), and greet(name). If you can write these cold, you can handle the exam.
  5. Practice reading error messages bottom-up. Several exam questions show an error and ask what caused it. Deliberately introduce bugs in your lab scripts — a typo in a variable name, a missing colon, dividing by zero — and study what Python says.
  6. Prepare YOUR own one-sheet. The act of compressing two units of material onto a single sheet is the studying. Do not skip this step.
  7. Time yourself tracing a loop on paper. Set a 3-minute timer and trace a small for loop by hand — write out each variable's value at each iteration. The exam will have at least one of these. Slow down, be systematic, don't rush past a step.
  8. Trace code by hand on paper. Loops on the exam will be small but ask "what does this print." Practice on paper with nested conditionals and simple accumulator patterns.

What makes a good cheatsheet

Your one double-sided sheet is the most valuable item you'll bring. Most students waste space on things they already know. Here is what actually earns points:

  • Syntax skeletons — the exact shape of an if/elif/else, a for loop, a while loop, a function definition. One line each. These are easy to forget under pressure.
  • Type conversion table — int(), float(), str(), bool() with one example each. Include a gotcha: int("3.5") raises a ValueError; use int(float("3.5")).
  • Operator precedence — a small table: **, then * / // %, then + -, then comparisons, then not, then and, then or.
  • Common error types — NameError, TypeError, ValueError, IndexError, ZeroDivisionError. One sentence each on what causes them.
  • Function anatomy — def, parameters, body indent, return. Note: a function without return returns None.
  • Git workflow — the four commands: git add, git commit -m "", git push, git pull. Order matters.
  • Truthy/falsy quick list — falsy: 0, 0.0, "", [], None, False. Everything else is truthy.

Do NOT waste space on things you know cold — variable assignment syntax, print(), the Python logo. Use the space for things that require active recall under pressure.

Sample Practice Problems

Work through these before the exam. They are representative of the style and difficulty of actual exam questions.

PRACTICE 1 · Code Trace
What does this code print? Trace it step by step.
total = 0
for i in range(1, 5):
    if i % 2 == 0:
        total += i
print(total)
Answer
6 — i takes values 1, 2, 3, 4. Only 2 and 4 are even. 2 + 4 = 6.
PRACTICE 2 · Bug Fix
This function is supposed to return the tax amount. Find and fix the bug.
def calculate_tax(price, rate):
    tax = price * rate
    print(tax)

result = calculate_tax(100, 0.07)
print(result + 100)
Answer
The function uses print(tax) instead of return tax, so it returns None. Fix: replace print(tax) with return tax.
PRACTICE 3 · Multiple Choice
What is the value of x after this code runs?
x = 10
def modify():
    x = 99
modify()
print(x)
  • a) 99
  • b) 10
  • c) None
  • d) NameError
Answer
b) 10 — The x = 99 inside the function creates a local variable. The global x is unchanged.
PRACTICE 4 · Error Reading
What type of error does this produce, and why?
price = "9.99"
total = price * 3
print(total)
Answer
TypeError — you cannot multiply a string by 3 using the * operator to get a numeric result. The fix is float(price) * 3. (Note: "9.99" * 3 actually produces "9.999.999.99" — string repetition — which may surprise you; if you wanted arithmetic, you needed float() first.)
PRACTICE 5 · Short Answer
Write a function apply_discount(price, pct) that returns the discounted price. If pct is 0.2, a price of 50 should return 40.0.
Answer
def apply_discount(price, pct):
    return price * (1 - pct)
