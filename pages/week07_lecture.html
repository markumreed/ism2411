Today's walkthrough

def, parameters, and return

A function is a chunk of code with a name. You define it once and call it many times.

def calculate_tax(price, rate):
    return price * rate

print(calculate_tax(100, 0.07))   # 7.0
print(calculate_tax(250, 0.08))   # 20.0

Reading a traceback

When Python crashes it prints a traceback. Read it bottom up. The bottom line tells you what went wrong; the lines above tell you where.

Traceback (most recent call last):
  File "sales.py", line 12, in <module>
    total = compute_total(price)
  File "sales.py", line 5, in compute_total
    return price * quantity
NameError: name 'quantity' is not defined

Debug First, Then Ask — the in-class exercise

You will get a script with 3 intentional bugs. Step 1: try to debug using only print() and the traceback. Step 2: paste only the error message into ChatGPT or Claude and ask what it means — NOT for the fix. Step 3: fix it yourself. Step 4: debrief.

The three modes of AI use

Explainer — "what does this error mean?" (low risk). Debugger — "where should I look?" (medium risk). Generator — "write this function for me" (high risk: you may not understand what you submitted, which means you cannot defend it on the exam).

Disclosure

Any time you use an AI tool on graded work, disclose it in a comment at the top of your file: # AI use: explained the TypeError on line 14. The disclosure protects you and is required by USF Academic Integrity policy.

In-class activities

  • Write a function calculate_total(price, quantity, discount) that returns the final amount.
  • Debug First, Then Ask — 20-minute exercise. Don't skip Step 1.
  • Canvas reflection (2-3 sentences): when did AI explanation help your learning? When could it have slowed you down?
