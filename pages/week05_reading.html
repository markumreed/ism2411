Why this matters

Until this week, your scripts ran straight through, top to bottom. Conditionals are the first time your code branches — does one thing if a condition is true, another if it isn't. Every spreadsheet IF formula you have ever written maps directly onto this.

What you will learn this week

  • if, elif, else
  • Writing decision logic
  • Nested conditionals — and when to avoid them
  • Business context: discount tiers, approval thresholds, flagging anomalies

Q1 · Code Trace
What is printed?
score = 85
if score >= 90:
    print('A')
elif score >= 80:
    print('B')
else:
    print('C')
Answer
B
Q2 · Multiple Choice
Which keyword introduces an alternative condition after if?
  • a) else
  • b) then
  • c) elif
  • d) elseif
Answer
elif
Q3 · Code Trace
What is the output?
discount = 0
total = 120
if total > 100:
    discount = 10
print(discount)
Answer
10
Q4 · Multiple Choice
What is the output of: x = 5; print(x > 3 and x < 10)?
  • a) True
  • b) False
  • c) Error
  • d) None
Answer
True
Q5 · Code Trace
What does this print?
approved = False
amount = 500
if amount > 1000:
    approved = True
print(approved)
Answer
False — 500 is not > 1000
Q6 · Multiple Choice
Which correctly flags a value as "high" when it exceeds 100?
  • a) if value = 100: flag = "high"
  • b) if value > 100: flag = "high"
  • c) if value > 100 then: flag = "high"
  • d) when value > 100: flag = "high"
Answer
if value > 100: flag = "high" — Use ==, not =, for comparison
