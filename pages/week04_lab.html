Revenue, Margin & Discount Calculator

Apply arithmetic, comparison, and logical operators to real business formulas.

Module 04 · Lab Unit 1

Lab Walkthrough

Exercises

EXERCISE 1 BASIC
Revenue
Given price = 50 and quantity = 12, compute and print revenue. Format with a thousands separator and two decimal places.
Expected: Revenue: $600.00
EXERCISE 2 BASIC
Margin
Given cost = 32 and price = 50, compute margin = (price - cost) / price. Format as a percentage with f"{margin:.1%}".
Expected: Margin: 36.0%
EXERCISE 3 BASIC
Two products
Create a small script comparing two products: which one has a higher margin? Use a comparison and print the answer as a boolean: product_a_wins = margin_a > margin_b.
Expected: prints margins for both products and a True/False for which is higher
EXERCISE 4 BASIC
Logical combo
Print whether a product is "premium": price > 100 AND margin > 0.3. Print True/False. Then change the values so one condition is met but not the other, and observe the result.
Expected: is_premium = True (or False depending on your values)
EXERCISE 5 BASIC
Common-mistake check
Try 17 / 5 and 17 // 5 and 17 % 5. Predict each output before running, then verify. For each one, write a sentence explaining what the operator does and when you would use it in a business calculation.
Expected: 3.4, 3, 2 — with written explanations
EXERCISE 6 INTERMEDIATE
Full pricing calculator
Write a script that asks the user for: product name, retail price, unit cost, quantity sold, and discount percentage (e.g., 10 for 10%). Compute and print: revenue (before discount), discounted revenue, gross margin, and net profit. Use f-strings with appropriate format specs for all numeric output.
Expected: a formatted report with five lines of output, all values correct for the input provided
EXERCISE 7 INTERMEDIATE
Packaging problem
A warehouse ships items in boxes of 24. Ask the user for the number of items ordered. Print: how many full boxes are needed, how many loose items remain, and whether the order fits in exactly full boxes (no remainder). Use //, %, and a comparison operator.
Expected: for 100 items — Full boxes: 4, Loose items: 4, Exact fit: False

Stretch Challenge

STRETCH A CHALLENGE
Break-even calculator
Write a script that asks for: fixed costs (rent, salaries, etc.), variable cost per unit, and retail price per unit. Compute the break-even quantity: the number of units needed to cover all costs. Formula: break_even = fixed_costs / (price - variable_cost). Use // to get the whole-unit answer and % to show the partial-unit remainder. Format all output professionally.
STRETCH B CHALLENGE
Operator precedence trap
Write five expressions where forgetting parentheses would give the wrong business answer. For each, show: (a) the expression without parentheses and its result, (b) the expression with correct parentheses and its result, (c) one sentence explaining which answer is correct and why. This exercise trains the professional habit of explicit parenthesization.

Reflection

Self-Assessment Questions
  1. In Exercise 6, what would happen if you forgot to divide the discount percentage by 100 (i.e., used 10 instead of 0.10)? Describe the error this would cause in the output — would Python crash, or would it produce a wrong answer silently?
  2. The modulo operator (%) might seem obscure, but Exercise 7 shows it solving a real logistics problem. Describe another business scenario where you would need to know both "how many full groups" and "how many are left over."
  3. Looking back at Modules 01–04, you have gone from "what is a computer" to writing a working pricing calculator that handles user input and produces formatted financial output. What concept from these four modules do you feel least confident about, and what would help you strengthen it?
