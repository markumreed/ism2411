INSTRUCTOR RESOURCE · COURSE DESIGN

Course Assembly Guide

This course is built as a library of independent modules. Professors can assemble any subset into a 6-week, 9-week, or full 16-week course without rewriting anything — just choose your modules and assign them in order.

How this works

Each module contains three components: a Reading page (concepts and quiz), a Lecture page (worked examples and in-class activities), and a Lab page (hands-on exercises). You can assign all three, or assign only what your format supports. A 90-minute workshop might use the Lecture and Lab only; an asynchronous course might rely on Reading and Lab.

Module Catalog

There are 14 teaching modules plus one midterm module and one capstone module, for 16 modules total. Each module is a complete, self-contained lesson. The table below shows every module, its key topics, approximate time, and which prerequisite modules should come before it.

6-Week Course — Intensive / Workshop Format

Designed for boot-camp sessions, professional development workshops, or an accelerated summer intensive. Each "week" is a full day (6–8 hours) or two 3-hour sessions. Six content blocks cover all essential Python-for-business skills; the capstone is a 2-hour mini-project on the final day.

Format assumptions

This format works best when students meet for full-day sessions (6–8 hours) or twice-weekly for 3 hours per session. DataCamp assignments from the 16-week course are optional here — use them as pre-work or evening practice if students have access.

Assessment — 6-Week Format

  • Block quizzes (3× informal) — short 5-question checks at end of Blocks 2, 3, and 4. Ungraded; used for pacing.
  • Lab submissions (each block) — students submit the lab notebook from each block. Graded on completion + correctness (rubric: does the code run without errors, does it produce the expected output).
  • Mini Capstone (Block 6) — 3-min presentation using the rubric at capstone_rubric.html, compressed to 3 business questions (not 5) and 3 charts (not 5).
What to skip and why

M08 Git: students need time to master Python logic first. Assign the Git reading as optional evening work. Submit labs via Canvas or email.
M09 Midterm: replace with an ungraded code challenge at end of Block 3 to gauge progress.
M12 Files: pandas handles CSV loading natively; the raw file I/O module adds value in the 16-week version but can be a 20-minute demo in this format.

9-Week Course — Trimester / Accelerated Semester

Designed for university trimesters, 10-week quarters (with finals week off), or half-semester intensive electives. Each "week" is one standard class week (2–3 class sessions, 4–6 total hours). All core concepts are covered; Git is included but the formal midterm is replaced with a formative assessment.

Format assumptions

Two or three class sessions per week, each 75–90 minutes. Labs are completed in class or assigned as take-home (1–2 hours). DataCamp assignments should be trimmed to the 4–5 most relevant courses (see below).

Tips for Combining Modules

When you merge two modules into one class session, here are the pairings that work best and the strategies that help:

M01 + M02 (Computer + CLI)

Morning: conceptual (CPU, memory, paths). Afternoon: hands-on terminal. These share the "file system" mental model so flow naturally. Use M01 as the motivating story, M02 as the practice.

M03 + M04 (Variables + Operators)

Variables give you things to work with; operators give you ways to compute with them. Teach assignment in the morning and expressions in the afternoon. Share one extended business scenario (price × quantity = revenue) across both.

M06 + M07 (Loops + Functions)

Loops introduce the idea of repetition; functions let you name a block of repeated logic. Teach a loop first, then show how wrapping it in a function makes it reusable. The debugging section of M07 works well as a standalone afternoon exercise.

M10 + M11 (Lists + Dicts)

Lists are ordered; dicts are labeled. Teach both on the same dataset (a product inventory) — first as a list, then restructured as a dict. Students see exactly when and why each structure is the right choice.

M13 + M14 (pandas intro + Cleaning)

Load the dataset in M13's lab, then immediately clean it in M14. Students stay in the same dataset, so there's no context switch — just progressively richer operations on the same DataFrame.

M14 + M15 (Cleaning + Visualization)

Clean the data in the morning; visualize it in the afternoon. Students see the direct payoff of cleaning work. This pairing works especially well in a capstone-prep session.

What to Skip (and What to Replace It With)

Skipping Git (M08)

What you lose: Professional workflow practice, portfolio-ready GitHub presence.
What to do instead: Have students save all work in a ism_course/ folder, zip it at the end, and submit via Canvas. Assign the Git reading as optional reading. You can introduce Git in the final week as a "you should know this exists" demo if time permits.

Skipping the Midterm (M09)

What you lose: A formal summative assessment at the halfway point.
What to do instead: Replace with a 30-minute lab checkpoint where students trace three short code snippets and explain one bug. Grade it pass/fail. This keeps momentum without losing the diagnostic value.

Skipping File I/O (M12)

What you lose: Raw file handling with Python's open() and the csv module.
What to do instead: Reduce M12 to a 20-minute lecture demo (show once, don't lab it). Students learn that open() exists; pandas' read_csv() handles the practical cases in M13. If you need to skip entirely, the pandas modules still work — they just assume the CSV is provided to students.

Choosing the Right Format

If you're uncertain which format to use, this quick decision guide helps:

  • Professional development or corporate training → 6-week intensive. Adults want to see results fast. Skip Git, compress foundations, spend most time on pandas.
  • 10-week quarter or trimester → 9-week course. Every core concept is covered at a reasonable pace. Treat the last week as overflow / catch-up if the class runs slow on loops or functions.
  • 16-week semester with general business students → Full course as designed. The pacing allows time for confusion and recovery. Weekly labs keep students practiced.
  • Graduate elective or MBA module → Consider starting at M03 (skip computer basics) and ending at M15 (skip or compress capstone). Add one module on APIs or SQL if your students already know Excel well.
Building your own sequence

You are not locked into these three formats. The only hard constraint is the dependency chain: M01 → M02 → M03 → M04 → M05 → M06 before anything that involves loops or iteration. Beyond that, M07 (Functions) and M10–M11 (Data Structures) can swap order, M08 (Git) can move anywhere, and the pandas block (M13–M15) is self-contained once students know lists and have a CSV to open. Mix, match, and trim as your course context demands.
