Why this matters

Up until now your scripts have handled one product at a time. Real businesses have thousands of products. Lists hold many things in order. Tuples hold a fixed record. Together they cover most of what you used to do with cells in Excel.

What you will learn this week

  • Creating, indexing, slicing, and modifying lists
  • Common list methods: append, remove, sort, len
  • When to use tuples vs. lists
  • Business context: a product inventory list

Q1 · Code Trace
What is the output?
items = ['pen', 'notebook', 'stapler']
print(items[1])
Answer
notebook — Lists are zero-indexed
Q2 · Code Trace
What is the output?
nums = [10, 20, 30, 40]
print(nums[-1])
Answer
40 — Index -1 returns the last element
Q3 · Multiple Choice
What does append() do to a list?
  • a) Removes the last item
  • b) Adds an item to the end
  • c) Sorts the list
  • d) Returns the length
Answer
Adds an item to the end
Q4 · Code Trace
What is the value of inventory?
inventory = ['apples', 'bananas']
inventory.append('oranges')
inventory.remove('bananas')
print(inventory)
Answer
['apples', 'oranges']
Q5 · Multiple Choice
What is the key difference between a list and a tuple?
  • a) Tuples hold more items
  • b) Lists use {}; tuples use []
  • c) Tuples are immutable; lists are mutable
  • d) Lists hold only strings
Answer
Tuples are immutable; lists are mutable
