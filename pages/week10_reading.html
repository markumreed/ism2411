Lists & Tuples

A product inventory: ['pen', 'notebook', 'stapler']. Lists are the workhorse collection in Python — ordered, flexible, and present in virtually every real-world script you'll write in this course.

Why this matters

Up until now your scripts have handled one product at a time. Real businesses have thousands of products, hundreds of customer orders, and daily sales figures that span months. Writing individual variables for each one is hopeless — product1, product2, product3... you'd need thousands of variable names. Lists solve this: a single variable holds as many items as you need, in order, and you can loop through all of them with a single for loop.

Think about what a spreadsheet column is. When a buyer at a retail chain downloads the product catalog from the ERP system, they get a column of SKUs, a column of prices, a column of quantities on hand. In Python, each of those columns is a list. The tools you learn this module — indexing, slicing, append, remove, sort — are the Python equivalents of the spreadsheet operations you've been doing manually for years. The difference is that you can now automate them across thousands of rows in milliseconds.

Tuples extend the idea further: they hold a fixed, ordered record that should never be changed. A GPS coordinate pair, an RGB color triplet, a database row — these are things you want to read and use, but never accidentally modify. Python enforces that guarantee with tuples. Together, lists and tuples cover the majority of what you previously did with cells and rows in Excel, but now in code that runs automatically.

Business context

A product inventory: ['pen', 'notebook', 'stapler']. Add new SKUs with .append(), retire discontinued ones with .remove(), and sort for a report with .sort() — all in a few lines of Python.

Core Concepts

List

An ordered, mutable collection in square brackets: products = ["Widget A", "Widget B", "Gadget C"]. Ordered means items have a fixed position. Mutable means you can add, remove, or change items after creation.

Index

The position of an item, starting at 0. products[0] is "Widget A". Negative indices count from the end: products[-1] is the last item. This trips up beginners — remember: zero-based, not one-based.

Slice

products[1:3] returns a new list with items at index 1 and 2 (stop index is excluded). products[:2] is "first two items." products[-2:] is "last two items."

List Methods

.append(item) adds to the end. .remove(item) deletes the first match. .sort() sorts in place. .reverse() reverses in place. .pop() removes and returns the last item. len(list) returns the count.

Tuple

An ordered, immutable collection in parentheses: coordinates = (40.71, -74.00). Once created, you cannot change it. Use tuples for data that must stay fixed — GPS coords, RGB values, database records.

Business Scenario

Imagine you're a buyer at a regional office supply chain. Each morning you receive a list of products that need restocking. You need to add new arrivals, remove items that are permanently discontinued, sort the list alphabetically for the warehouse team, and compute how many SKUs are currently active. Here is how that looks in Python:

# Morning inventory workflow
inventory = ["stapler", "notebook", "pen", "tape", "marker"]

# New shipment arrived — add to inventory
inventory.append("envelope")

# Discontinued product — remove it
inventory.remove("tape")

# Sort for the warehouse report
inventory.sort()

# How many active SKUs?
print(f"Active SKUs: {len(inventory)}")
print(inventory)

# Output:
# Active SKUs: 5
# ['envelope', 'marker', 'notebook', 'pen', 'stapler']

Notice that the list is modified in place by .sort() and the other methods. There is no need to assign the result back to a variable — the original list changes directly.

Key Vocabulary

  • List — an ordered, mutable sequence of items enclosed in square brackets
  • Tuple — an ordered, immutable sequence enclosed in parentheses; cannot be changed after creation
  • Index — the integer position of an element in a sequence, starting at 0
  • Slice — a sub-sequence extracted using [start:stop] notation; stop index is excluded
  • Mutable — an object whose contents can be changed after creation (lists are mutable)
  • Immutable — an object whose contents cannot be changed after creation (tuples and strings are immutable)
  • Append — the .append(item) method that adds a single item to the end of a list
  • In-place operation — a method that modifies the original object rather than returning a new one; .sort() and .reverse() are in-place
Common Misconceptions

Off-by-one indexing: Lists are zero-indexed. products[1] is the second item, not the first. On exam questions that ask "what does items[1] return," remember to count from zero.

Sort returns None: sorted_list = my_list.sort() assigns None to sorted_list because .sort() modifies the list in place and returns nothing. Use sorted(my_list) if you need a new sorted list without changing the original.

Tuples vs. lists syntax: Lists use []; tuples use (). A one-item tuple requires a trailing comma: single = (42,) — without the comma, Python treats (42) as just the integer 42 in parentheses.

Looking Ahead — Module 11

Next module you'll meet dictionaries — the next step up in data organization. Where a list says "here are five products in order," a dictionary says "here is one product and here is everything I know about it: its name, price, SKU, and stock level." Q1 · Code Trace
What is the output?
items = ['pen', 'notebook', 'stapler']
print(items[1])
Answer
notebook — Lists are zero-indexed; index 1 is the second item

What is the output?
nums = [10, 20, 30, 40]
print(nums[-1])
Answer
40 — Index -1 returns the last element
Q3 · Multiple Choice
What does append() do to a list?
  • a) Removes the last item
  • b) Adds an item to the end
  • c) Sorts the list
  • d) Returns the length
Answer
b) Adds an item to the end
Q4 · Code Trace
What is the value of inventory?
inventory = ['apples', 'bananas']
inventory.append('oranges')
inventory.remove('bananas')
print(inventory)
Answer
['apples', 'oranges']
Q5 · Multiple Choice
What is the key difference between a list and a tuple?
  • a) Tuples hold more items
  • b) Lists use {}; tuples use []
  • c) Tuples are immutable; lists are mutable
  • d) Lists hold only strings
Answer
c) Tuples are immutable; lists are mutable
Q6 · Code Trace
What is the output?
coords = (40.7128, -74.0060)

2
Q7 · Code Trace
What does this print?
sales = [100, 250, 75, 480, 200]
print(sales[1:4])
Answer
[250, 75, 480] — Slice includes index 1, 2, 3; excludes index 4
Q8 · Multiple Choice
What happens when you call my_list.sort()?
  • a) Returns a new sorted list; original unchanged
  • b) Sorts the list in place; returns None
  • c) Returns the sorted list
  • d) Raises a TypeError
Answer
b) Sorts the list in place; returns None
Q9 · Bug Fix
This code has a bug. What is wrong and how do you fix it?
products = ["Widget A", "Widget B", "Gadget C"]
first = products[1]
print("First product:", first)
Answer
The bug is using index 1 instead of 0. Index 1 returns "Widget B", not "Widget A". Fix: first = products[0].
