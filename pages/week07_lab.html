Exercises

EXERCISE 1
BASIC
calculate_tax
Define def calculate_tax(price, rate): return price * rate. Call it with (100, 0.07) and (250, 0.08). Verify the outputs match expectations before moving on.

EXPECTED OUTPUT

Tax on $100 at 7%: $7.00
Tax on $250 at 8%: $20.00
EXERCISE 2
BASIC
apply_discount
Define def apply_discount(price, tier) that returns the discounted price. Use if/elif on tier: "gold" → 15%, "silver" → 10%, "bronze" → 5%, else 0%. Test all four tiers with price = $200.

EXPECTED OUTPUT

gold:   $170.00
silver: $180.00
bronze: $190.00
none:   $200.00
EXERCISE 3
INTERMEDIATE
Compose them
Compute the final price: starting at $200, apply gold discount, then add 7% tax. Use both your functions together. Then write a third function final_price(price, tier, tax_rate) that calls both and returns the result.

EXPECTED OUTPUT

After gold discount: $170.00
After tax: $181.90
final_price(200, "gold", 0.07) = $181.90
EXERCISE 4
INTERMEDIATE
Loop + function
Given a list of (price, tier) tuples, use a for loop and your final_price function to print a complete sales report for each item.

STARTER CODE

orders = [(200, "gold"), (150, "silver"), (80, "bronze"), (500, "none")]
for price, tier in orders:
    total = final_price(price, tier, 0.07)
    print(f"Price: ${price} | Tier: {tier} | Final: ${total:.2f}")
EXERCISE 5
INTERMEDIATE
Debug First, Then Ask
You will be given broken_sales.py with 3 intentional bugs. Follow the four-step protocol:
  1. 5 min — Solo debug: Add print() statements to trace variable values. Read the traceback. Write down what you think each bug is before fixing anything.
  2. 5 min — AI Explainer: Paste only the error message (not your code) into ChatGPT or Claude. Ask "what does this error mean?" Do not ask for the fix.
  3. 5 min — Fix it yourself: Use the AI's explanation as a guide. Write the fix in your own words as a comment before implementing it.
  4. 5 min — Class debrief: Share what you found. Did the AI's explanation match the actual bug?
EXERCISE 6
INTERMEDIATE
Scope experiment
Run the code below and predict what happens. Then run it and explain why the output is what it is. Add a comment in the file explaining scope in your own words.
x = 100   # global variable

def double_it():
    x = 999   # local variable — different from global x!
    return x * 2

print(x)           # what prints here?
print(double_it()) # what prints here?
print(x)           # has x changed?

Stretch Challenge

STRETCH 1 — CHALLENGE
Summarize a list with a function
Write a function summarize(sales_list) that takes a list of numbers and returns a dictionary with keys: "total", "count", "average", "max", "min". Call it and print each key/value pair. Hint: use a loop inside the function and the accumulator pattern.
STRETCH 2 — CHALLENGE
Default parameters
Rewrite calculate_tax(price, rate=0.07) with a default rate of 7%. Verify that calculate_tax(100) returns 7.0 and calculate_tax(100, 0.1) returns 10.0. Then write a docstring for the function — a triple-quoted string immediately after the def line that explains what it does, its parameters, and its return value.
