What you'll learn

Control Flow & Structure is where Python starts to feel like a real tool rather than a calculator. You will learn loops so you can process every row in a dataset without writing the same code hundreds of times, functions so you can name a useful piece of logic and reuse it reliably, and Git so your work is saved, auditable, and shareable. Module 7 also introduces rubber duck debugging (Hunt & Thomas, The Pragmatic Programmer, 1999) — explaining a bug out loud to an inanimate object before asking for help — alongside the "Debug First, Then Ask" AI literacy framework that shapes how you use coding assistants responsibly. By the end of Unit 2, you can write programs that do meaningful repetitive work and can collaborate on code professionally.

Unit Outcomes

After completing Unit 2, you will be able to:

  • Write for loops and while loops to process lists and ranges of data without manual repetition
  • Define and call functions with parameters, arguments, and return values
  • Read a Python traceback and identify the line, error type, and likely cause
  • Use rubber duck debugging to articulate a bug before reaching for external help
  • Use Git to init, add, commit, and push code to a GitHub repository
  • Apply the "Debug First, Then Ask" framework when working with AI coding assistants

Module Connections

Unit 2 builds directly on the variables, data types, and conditionals from Unit 1 — loops and functions are meaningless without the expressions and decision logic from Modules 3–5. Unit 2 in turn prepares you for Unit 3: the data structures (lists, dictionaries, files) covered in Modules 10–12 are most useful when you can iterate over them with loops and process them with reusable functions. Git, introduced in Module 8, becomes increasingly important from Module 13 forward, when you begin submitting labs as GitHub repository URLs.
