Loops

Loops are the reason we use programming for data work. Anything you want to do to one row of data, a loop lets you do to a million rows.

Today's walkthrough

for over a list

The most common loop pattern: iterate through a collection one item at a time. Python's for loop is designed for this — you do not need an index, a counter, or a length check. Just name the variable that will hold each item, and Python handles the rest.

The variable name you choose (sale, item, row) should describe what each element represents. It is a temporary variable — it only exists while the loop is running, and it gets overwritten each iteration. After the loop, it holds the last value it was assigned.

Below is the accumulator pattern in its simplest form. We initialize total to zero before the loop, update it inside the loop, and read the final result after the loop. This three-step pattern — initialize, loop, update — applies to sums, counts, maximums, minimums, and string concatenation.

sales = [100, 250, 75, 480, 200]
total = 0
for sale in sales:
    total += sale
print(total)  # 1105

for with range()

When you want to repeat something a fixed number of times, or generate a numeric sequence, use range(). It takes up to three arguments: start, stop (excluded), and step. If you only give one argument, it starts at 0.

A common use in business: generating row numbers, creating numbered reports, or processing data in fixed-size batches. range() is lazy — it does not store all the numbers in memory at once, which is why it works fine even for range(10_000_000).

You can combine range() with len() to get index-based access to a list: for i in range(len(sales)): gives you both the index i and the value sales[i]. This is less Pythonic than a direct for loop, but necessary when you need the index itself.

for i in range(5):
    print(i)
# 0 1 2 3 4

for i in range(2, 6):
    print(i)
# 2 3 4 5

# Step argument — every other number
for i in range(0, 10, 2):
    print(i)
# 0 2 4 6 8

while loops

A while loop keeps going as long as a condition is True. Use it when you do not know in advance how many iterations you need — for example, reading user input until they enter a valid value, or processing records until you reach the end of a file.

The critical rule with while loops: something inside the loop must eventually make the condition False, or the loop runs forever. This is called an infinite loop, and it is one of the most common bugs beginners write. The fix is always to ensure the loop variable is updated inside the body.

A useful mental check before writing a while loop: "What will change between iterations? How will the condition eventually become False?" If you cannot answer both questions, you probably need to restructure the logic.

count = 0
while count < 4:
    print("hello")
    count += 1  # without this line: infinite loop!

break and continue

break exits the loop immediately — the remaining items are not processed, and Python jumps to the first line after the loop. It is useful for early exit: once you have found what you are looking for, there is no point continuing.

continue skips the rest of the current iteration and moves to the next item. It is useful for filtering: when a condition means the current item should be ignored, continue keeps the loop going while bypassing the processing code for that item.

Both should be used sparingly. A loop with many breaks and continues becomes hard to reason about. Often, restructuring the logic with an if/else is cleaner and easier to read.

for sale in sales:
    if sale < 0:
        continue   # skip negatives
    if sale > 10000:
        break      # stop on huge outlier
    print(f"Processing sale: ${sale}")

The pattern you will use 100 times this semester

Initialize, loop, update. This is not just a Python pattern — it is the fundamental structure of any aggregation over a dataset. SQL's GROUP BY, Excel's SUM, pandas' groupby are all abstractions of this same idea.

The key insight: you cannot get the result in one step. You build it incrementally, one item at a time. The accumulator variable is your running total, running count, or running maximum — it carries state between iterations.

total = 0          # initialize
for x in data:     # loop
    total += x     # update
print(total)       # result

Detailed Code Walkthrough

Walkthrough 1: Full sales report in one pass

This example computes four statistics in a single loop. We use four accumulators simultaneously, then do the calculations that require all iterations to be complete (average) after the loop ends.

sales = [120, 80, 250, 175, 90, 410, 60, 215]

total = 0           # line 3: will hold the sum
count = 0           # line 4: will hold the number of transactions
current_max = 0     # line 5: will hold the largest value
above_100 = 0       # line 6: will count transactions over $100

for sale in sales:              # line 8: iterate over each sale
    total += sale               # line 9: add to running total
    count += 1                  # line 10: increment counter
    if sale > current_max:      # line 11: is this the new max?
        current_max = sale      # line 12: if so, update max
    if sale > 100:              # line 13: filter check
        above_100 += 1          # line 14: increment filtered count

average = total / count         # line 16: calculated AFTER loop completes

print(f"Total:    ${total}")
print(f"Count:    {count}")
print(f"Average:  ${average:.2f}")
print(f"Max:      ${current_max}")
print(f"Over $100: {above_100}")

Walkthrough 2: Finding the first match with break

This pattern is common in business systems: scan through records and stop as soon as you find one that meets a condition. The found flag tells you whether the loop ended because of break or because it ran out of items.

transactions = [50, 120, 300, 80, 25, 600, 95]
threshold = 500
found = False

for t in transactions:
    if t >= threshold:
        print(f"Alert: large transaction found — ${t}")
        found = True
        break           # stop after finding the first one

if not found:
    print("No transactions exceed the threshold.")

Common Errors

ERROR 1 — Infinite while loop
# WRONG — count never changes, loop runs forever
count = 0
while count < 5:
    print("hello")
    # forgot: count += 1

# RIGHT
count = 0
while count < 5:
    print("hello")
    count += 1
ERROR 2 — Accumulator initialized inside the loop
# WRONG — total resets to 0 every iteration!
for sale in sales:
    total = 0       # this resets every time — bug!
    total += sale
print(total)        # only shows the last sale

# RIGHT — initialize before the loop
total = 0
for sale in sales:
    total += sale
print(total)
ERROR 3 — Off-by-one in range
# WRONG — range(5) gives 0,1,2,3,4 (not 1-5)
for i in range(5):
    print(i)   # prints 0,1,2,3,4

# If you want 1 through 5:
for i in range(1, 6):
    print(i)   # prints 1,2,3,4,5

Key Takeaways
  • The accumulator pattern (initialize → loop → update) is the foundation of all aggregation in data work.
  • Use for when you are iterating over a collection; use while when the stopping condition depends on runtime state.
  • range(stop) starts at 0 and excludes stop — range(5) gives 0, 1, 2, 3, 4.
  • break exits the loop; continue skips to the next iteration — they are not interchangeable.
  • Always initialize your accumulator variables before the loop begins, not inside it.
