AI Use Policy

AI tools are part of how modern developers work. This course teaches you to use them deliberately — knowing when they help, when they hurt, and how to disclose what you used. Three modes are approved. Each has rules.

The short version: You may use ChatGPT, Claude, GitHub Copilot, or similar tools for labs and homework. You must (1) understand every line you submit, (2) disclose what you used and how, and (3) follow the mode-specific rules below. On the midterm exam, AI tools are not permitted.

Why we have a policy

You are going to use these tools in your career. The companies that hire USF graduates expect it. But there's a difference between using AI well — to learn faster, debug smarter, automate the boring parts — and using it badly, where you ship code you don't understand and can't defend.

This policy gives you three approved modes. Each one teaches a different skill.

Mode 1 — Explainer

You read code or output you don't understand, and ask the AI to explain it.

Allowed: Pasting a code snippet, error message, or function from the textbook and asking "what does this do" or "why does this happen."

Example prompt: "Here's a Python error: TypeError: unsupported operand type(s) for +: 'int' and 'str'. What does this mean and what causes it?"

Why it's safe: You're not generating new code, you're learning. This is exactly what a good tutor does.

Mode 2 — Debugger

You wrote code, it doesn't work, you ask the AI to help find the bug.

Allowed: Pasting your code with the error and asking what's wrong. Asking the AI to explain why a fix works.

Required step before asking: Try to find the bug yourself for at least 10 minutes. Print the variables. Read the error. Check the line number. If you still can't find it, then ask. We call this "Debug First, Then Ask" — there's an exercise on this in Week 7.

Why we require the wait: Most bugs in your career will not have an AI to fix them quickly. The skill of staring at broken code and reasoning your way out is the actual job. AI is a co-pilot, not a pilot.

Mode 3 — Generator

You ask the AI to write code for you.

Allowed only under these conditions:

  1. You can explain every line of the generated code in your own words. If asked in class, you should be able to walk through it.
  2. You disclose generator use in your submission (see disclosure section below).
  3. The generated code is for boilerplate or syntax you've already been taught — not for the core problem-solving step of the lab.

Example of acceptable generator use: "Write a function signature in Python that takes a list of dicts and returns the average of the 'price' key." — You then complete the body yourself and verify.

Example of unacceptable generator use: "Solve this week's lab for me." — You skipped the learning. You won't pass the midterm.

Disclosure

Every lab submission must include a one-line disclosure at the top of your main file, as a Python comment:

# AI USE: None
# - or -
# AI USE: Explainer (asked about TypeError on line 14)
# - or -
# AI USE: Debugger (helped fix loop in problem 3)
# - or -
# AI USE: Generator (generated the file-reading skeleton in problem 2, I wrote the logic)

Honest disclosure costs nothing. Undisclosed use, if discovered, is academic dishonesty.

What's not allowed

  • On the midterm exam (Week 9): No AI tools of any kind. Closed-resource environment.
  • On the capstone (Week 16): Generator mode is restricted to syntax-only — never for the analytical decisions (which columns to compare, what story the data tells, which chart to choose). Disclosure required.
  • Pasting other students' code into an AI tool to "rewrite" it and calling it your own. That's plagiarism with extra steps.
  • Submitting code you cannot explain. If you can't read your own submission aloud and describe what each line does, you're not ready to submit.

Why this matters in your career

The graduates who do well are the ones who can verify AI output. The ones who get fired are the ones who shipped a number to a VP that turned out to be wrong because they trusted the AI without checking. This course is your training ground for that judgment.
