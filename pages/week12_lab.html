Read a Sales CSV, Write a Cleaned Report

Open a CSV file with the csv module, process the rows, filter for high-value orders, and write a cleaned output file. This is your first complete ETL (Extract, Transform, Load) pipeline.

Module 12 · Lab Unit 3 GitHub submission

Exercises

EXERCISE 1 · Setup
Get the file
Download sales.csv (provided on Canvas). Place it in week12/data/sales.csv. Confirm the file structure: it should have a header row with columns product, price, quantity and at least 10 data rows. If you don't have it yet, create a sales.csv yourself with 10 rows of sample data.
Expected: file exists at the relative path; opening it with Python produces no error
EXERCISE 2 · Beginner
Read and print raw rows
Use csv.reader with a with block to open and iterate. Print every row including the header. Observe that all values are strings.
Expected: each row prints as a list, e.g. ['Widget A', '9.99', '50']
EXERCISE 3 · Beginner
Skip the header
Use next(reader) to skip the header row. Store the header in a variable and print it separately before the data rows. Confirm your loop now starts at the first data row.
Expected: header printed once, then data rows printed without the header
EXERCISE 4 · Intermediate
Compute totals
For each row, compute price * quantity (remembering to convert types), and print each line formatted as "Widget A: 50 units × $9.99 = $499.50". After the loop, print the overall total revenue across all rows.
Expected: one formatted line per product, then total revenue at the bottom
EXERCISE 5 · Intermediate
Filter and write
Write a new CSV called week12/data/cleaned.csv containing only orders with total > $100. Include a header row with columns: Product, Price, Qty, Total. Use newline="" in the open() call. Print how many rows were written vs. how many were in the original.
Expected: cleaned.csv created; count of qualifying rows printed
EXERCISE 6 · Intermediate
csv.DictReader
Re-read the same sales.csv file using csv.DictReader. Access row["product"], row["price"], row["quantity"] by name instead of by index. Confirm you get the same totals as Exercise 4. Add a 2-line comment comparing the two approaches — when would you prefer each one?
Expected: same totals as Exercise 4; comment in the code comparing reader vs DictReader
EXERCISE 7 · Intermediate
Sales summary by product
Read the CSV and build a summary dictionary {product_name: total_revenue}. If a product appears in multiple rows, accumulate the totals. Print the summary sorted by product name. This combines what you learned about dictionaries (Module 11) with file I/O (this module).
Expected: one line per unique product showing accumulated revenue
EXERCISE 8 · Advanced
Append mode log
Create a text file week12/data/run_log.txt. Each time your script runs, append one line to this file with the current date/time and the total revenue computed from the CSV. Run your script three times and open the log file — it should have three entries, not one. Use from datetime import datetime to get the current timestamp.
Expected: run_log.txt has one new line per script execution, accumulating over runs

Stretch Challenge

Extend Exercise 7 to write the product summary as a new CSV file called summary.csv with columns Product and TotalRevenue, sorted by revenue descending (highest revenue first). Then open summary.csv in Excel or a text editor to verify the output is correct.

Bonus: use csv.DictWriter instead of csv.writer for the output — it makes the column-to-value mapping explicit and is safer for files with many columns.

Reflection

Before you submit

In a Python comment at the top of your file, answer these two questions in 2-3 sentences each:

  1. You've now completed a full ETL pipeline: read a CSV, transform the data, write a new CSV. How does this compare to doing the same task in Excel? What can Python do that Excel can't do easily, and vice versa?
  2. Exercise 8 showed you append mode. Think of a real business process that would benefit from an append-mode log file — describe it and explain why append mode is the right tool.
Submission

Submit csv_practice.py + the cleaned CSV and any other output files via GitHub (week12/ folder), then submit the repo URL to Canvas.
