EXERCISE 2
BASIC
New repo
On github.com, click New repository. Name it ism2411. Make it Public. Add a README. Clone it to your local machine.

COMMANDS

# Clone the repo to your local machine
git clone https://github.com/YOURUSERNAME/ism2411.git

# Navigate into it
cd ism2411

# Verify Git is tracking it
git status
EXERCISE 3
BASIC
Add a .gitignore
Create a .gitignore file in your repo root. Add common Python and OS exclusions, then commit and push it before adding any other files.

STARTER .gitignore CONTENT

# Python
__pycache__/
*.pyc
*.pyo
.env
*.egg-info/

# OS
.DS_Store
Thumbs.db

# Editors
.vscode/
.idea/

COMMIT COMMANDS

git add .gitignore
git commit -m "add .gitignore for Python and OS files"
git push
EXERCISE 4
INTERMEDIATE
Add Module 7 work
Copy your functions.py from Module 7 into a new folder ism2411/week07/. Stage, commit, and push it with a descriptive message.

COMMANDS

# From inside ism2411/:
mkdir week07
cp /path/to/your/functions.py week07/

git status            # confirm Git sees the new file
git add week07/
git commit -m "add module 7 functions: calculate_tax, apply_discount, final_price"
git push

VERIFICATION

# Visit https://github.com/YOURUSERNAME/ism2411
# You should see the week07/ folder and your commit message
EXERCISE 5
INTERMEDIATE
Write a README
Open README.md. Add: a title (# ISM2411 — Python for Business), your name, semester, and a short description of the repo. Use markdown headings and bullet lists. Commit and push.

README TEMPLATE

# ISM2411 — Python for Business

**Name:** Your Name Here
**Semester:** Fall 2025
**Instructor:** [Course Instructor]

## About This Repo

This repository contains my weekly lab submissions for ISM2411.
Each folder corresponds to one module.

## Modules

- `week05/` — Conditionals: tiered discount calculator
- `week06/` — Loops: sales report with sum, average, max
- `week07/` — Functions: calculate_tax, apply_discount
- `week08/` — Git: first GitHub submission

## How to Run

Each script is standalone. Open a terminal and run:
```
python week07/functions.py
```
EXERCISE 6
INTERMEDIATE
Three commits
Make at least 3 separate commits this module with descriptive messages. Each commit should be for a distinct, coherent change — not "update everything at once." Run git log --oneline to verify all three appear. Confirm each shows up on github.com.

EXPECTED git log OUTPUT

a3f1b2c add module 7 functions: calculate_tax, apply_discount, final_price
9d4e1a0 add .gitignore for Python and OS files
f7c2b3e add README with module listing and run instructions

Stretch Challenge

STRETCH 1 — CHALLENGE
Back-fill earlier modules
Add your work from Modules 5 and 6 (discount.py and sales_loop.py) to the appropriate folders in your repo. Write a descriptive commit message for each. Then update the README to include those modules. Practice the full git add → commit → push cycle for each addition separately — do not lump everything into one commit.
STRETCH 2 — CHALLENGE
Simulate a bug and rollback
In your functions.py, deliberately break one of your functions (e.g., change a return statement to return the wrong value). Commit the broken version with the message "introduce bug for rollback exercise". Then use git revert or git checkout to restore the working version and commit the fix with a message that describes what the bug was.

Reflection

Self-Assessment

1. Run git log --oneline in your repo and copy the output here (as a comment in your submitted file). Look at your commit messages — if you were hiring someone and saw this history, what would it tell you about how they work?

2. Describe in 2–3 sentences how using Git changes (or should change) how you approach making edits to a working script. What would you do differently now compared to before this module?

3. What is one thing you would add to your .gitignore that is specific to your own machine or workflow that was not in the starter template? Why should that file not be tracked?
