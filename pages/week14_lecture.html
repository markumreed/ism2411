Data Cleaning & Descriptive Stats

Real data is messy. Missing values, wrong types, weird column names. Today we learn the cleaning workflow that turns a raw dump into something you can actually analyze.

Today's walkthrough

Finding missing values

The first thing you do with any new dataset is check for missing values. A missing value in pandas is called NaN (Not a Number). It can appear in any column type — numeric, string, or date — and it propagates silently: NaN + 5 = NaN, NaN == "South" = False. If you don't catch it early, it will corrupt your groupby results and your visualizations without any error message.

The standard diagnostic is df.isnull().sum(). This returns a count of NaN values per column. Pair it with df.info(), which shows the non-null count alongside the dtype. When the non-null count is less than the total row count, you have missing data.

df.isnull().sum()       # count missing per column
df.info()               # types + non-null counts

# Percentage missing per column:
(df.isnull().sum() / len(df) * 100).round(1)

Dropping or filling

Once you know where the missing values are, you have two strategies: drop the rows that have them, or fill them with something. The right choice depends on the business context. If a row is missing the order ID — the essential identifier — there is nothing useful to infer; drop it. If a numeric column like quantity is missing for 2% of rows and you need to compute totals, filling with the column mean is a reasonable approximation that preserves the other data in those rows.

Always document your choice with a comment. "I dropped rows with missing order_id because there is no way to attribute the revenue." "I filled missing quantity with the mean because the analysis is at the aggregate level and 2% imputation has negligible effect." Your future self (and your instructor) will thank you.

df = df.dropna()                          # drop rows with ANY missing value
df = df.dropna(subset=["price"])          # drop only if price is missing

df["price"] = df["price"].fillna(0)        # fill with 0
df["price"] = df["price"].fillna(df["price"].mean())   # fill with the column mean

Fixing column names

Column names with spaces, capital letters, or special characters are a constant source of pain. You cannot write df.Order Date without an error. You have to use df["Order Date"] every single time. Standardizing to lowercase snake_case in one line at the top of your cleaning script eliminates this friction for the rest of the analysis. It also makes your code more readable and more predictable.

df.columns = df.columns.str.lower().str.replace(" ", "_")
# "Order Date" → "order_date"
# "Product Category" → "product_category"
# "Customer ID" → "customer_id"

Fixing data types

pandas infers data types when loading a CSV. If a numeric column has even one non-numeric value ("$1,200", "N/A", an empty cell), pandas loads the entire column as object (string). You must strip the non-numeric characters and then cast the type. For dates, use pd.to_datetime() with errors="coerce" to convert bad date strings to NaN rather than crashing.

df["order_date"] = pd.to_datetime(df["order_date"], errors="coerce")
df["price"] = df["price"].str.replace("$", "", regex=False).astype(float)
df["quantity"] = df["quantity"].astype(int)

Descriptive stats

df.describe() is your data quality X-ray. Run it after cleaning and look for: a min value that should not be negative; a max value that is 10× the 75th percentile (outlier); a count less than the total rows (still missing data); a mean and median far apart (skewed distribution). Each of these is a flag to investigate before proceeding to analysis.

df.describe()

df["revenue"].mean()
df["revenue"].median()
df["revenue"].std()
df["revenue"].min(), df["revenue"].max()

# Value counts for categorical columns:
df["region"].value_counts()

Documenting what you cleaned

In a real workflow — and especially in the capstone — you keep a short log of every cleaning decision. This can be a markdown cell in Jupyter or a series of comments in a Python script. The goal is that someone reading your code can understand not just what you did but why. "Dropped 47 rows with missing order_date — these appear to be system-generated test orders" is far more informative than just df = df.dropna(subset=["order_date"]).

Live Coding Walkthrough

import pandas as pd

# ── Load ──────────────────────────────────────────────────────
df = pd.read_csv("data/messy_sales.csv")
print("Raw shape:", df.shape)

# ── Inspect ───────────────────────────────────────────────────
print(df.info())
print("\nMissing values:\n", df.isnull().sum())

# ── Fix column names ──────────────────────────────────────────
df.columns = df.columns.str.lower().str.replace(" ", "_")
print("\nCleaned columns:", df.columns.tolist())

# ── Handle missing values ─────────────────────────────────────
# Drop rows with no customer_id (can't attribute revenue)
before = len(df)
df = df.dropna(subset=["customer_id"])
print(f"Dropped {before - len(df)} rows missing customer_id")

# Fill missing quantity with mean (low % missing, aggregate analysis)
df["quantity"] = df["quantity"].fillna(df["quantity"].mean())

# ── Fix types ─────────────────────────────────────────────────
df["price"] = df["price"].str.replace("$", "", regex=False).astype(float)
df["order_date"] = pd.to_datetime(df["order_date"], errors="coerce")
bad_dates = df["order_date"].isnull().sum()
print(f"Unparseable dates: {bad_dates}")

# ── Check duplicates ──────────────────────────────────────────
print(f"Duplicate rows: {df.duplicated().sum()}")
df = df.drop_duplicates()

# ── Describe ──────────────────────────────────────────────────
print("\nDescriptive stats after cleaning:")
print(df.describe())

# ── Save ──────────────────────────────────────────────────────
df.to_csv("data/clean_sales.csv", index=False)
print("\nSaved clean_sales.csv")

Common Errors

Error 1: TypeError when computing stats on a string column

# Error: TypeError: unsupported operand type(s) for +: 'float' and 'str'
df["revenue"].mean()  # fails if revenue was loaded as object

# Fix: convert the column first
df["revenue"] = df["revenue"].astype(float)
df["revenue"].mean()  # now works

Error 2: Forgetting to assign after fillna/dropna

# WRONG — df is unchanged
df["price"].fillna(0)
print(df["price"].isnull().sum())  # still shows missing values

# RIGHT — assign back
df["price"] = df["price"].fillna(0)
print(df["price"].isnull().sum())  # 0

Error 3: pd.to_datetime crashing on bad dates

# WRONG — crashes if any date is unparseable
df["date"] = pd.to_datetime(df["date"])

# RIGHT — coerce bad dates to NaT (Not a Time) instead of crashing
df["date"] = pd.to_datetime(df["date"], errors="coerce")
print(df["date"].isnull().sum())  # see how many failed

Key Takeaways
  • Always run df.isnull().sum() and df.info() immediately after loading any new dataset.
  • Drop rows when the missing column is essential and cannot be inferred; fill when the column is supplementary and a reasonable estimate can be made.
  • Standardize column names to snake_case in one line at the top of every cleaning script.
  • Use errors="coerce" with pd.to_datetime() to handle bad dates gracefully.
  • Document every cleaning decision as a comment — your future self and your team will depend on it.
