Data Cleaning & Descriptive Stats

Real data is messy. Missing values, wrong types, weird column names. Today we learn the cleaning workflow that turns a raw dump into something you can actually analyze.

Today's walkthrough

Finding missing values

df.isnull().sum()       # count missing per column
df.info()               # types + non-null counts

Dropping or filling

Two strategies: drop rows with missing data, or fill them with something sensible.

df = df.dropna()                          # drop rows with ANY missing value
df = df.dropna(subset=["price"])          # drop only if price is missing

df["price"] = df["price"].fillna(0)        # fill with 0
df["price"] = df["price"].fillna(df["price"].mean())   # fill with the column mean

Fixing column names

Lowercase, no spaces, snake_case — your future self will thank you.

df.columns = df.columns.str.lower().str.replace(" ", "_")
# "Order Date" → "order_date"
# "Product Category" → "product_category"

Fixing data types

df["order_date"] = pd.to_datetime(df["order_date"])
df["price"] = df["price"].astype(float)
df["quantity"] = df["quantity"].astype(int)

Descriptive stats

describe() gives you a fast summary of every numeric column.

df.describe()

df["revenue"].mean()
df["revenue"].median()
df["revenue"].std()

Documenting what you cleaned

In a real workflow you keep a short log of every cleaning decision. We do this in our capstone too.

In-class activities

  • Run .isnull().sum() on a messy CSV. Decide what to drop vs. fill — and write down why.
  • Standardize the column names to snake_case in one line.
  • Use describe() to get a summary; explain in one sentence what each column tells you.
