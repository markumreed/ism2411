Setup & Installation
Python says “command not found” or “No such file or directory” on Mac

On Mac, the correct command is python3, not python. The bare python command either does not exist or still points to the old Python 2 that shipped with older macOS versions.

Fix: Always use python3 in the terminal:

python3 --version

VS Code handles this automatically once the Python extension is installed and the interpreter is selected — you only need python3 when typing commands directly in the terminal.

“python is not recognized as an internal or external command” on Windows

This means Python was installed without being added to your PATH. On the Python installer’s first screen, the “Add Python to PATH” checkbox at the bottom was not checked.

Fix:

  • Open Settings → Apps and uninstall Python
  • Re-download from python.org/downloads
  • On the first screen of the installer, check “Add Python 3.12 to PATH” before clicking anything else
  • Open a new PowerShell window and run python --version to confirm
VS Code says “No Python interpreter selected”

VS Code needs to know which Python installation to use. This prompt appears the first time you open a .py file or try to run one.

Fix: Press ⌘Shift+P (Mac) or Ctrl+Shift+P (Windows) to open the Command Palette. Type Python: Select Interpreter and press Enter. Choose the entry that shows Python 3.10 or higher. The version number will appear in the bottom-right corner of VS Code whenever a .py file is open.

The run button is missing, or right-clicking doesn’t show “Run Python File in Terminal”

The Run Python File in Terminal option only appears when the Python extension is installed and enabled in VS Code.

Fix:

  • Open Extensions with ⌘Shift+X (Mac) or Ctrl+Shift+X (Windows)
  • Search for Python
  • Confirm the Microsoft extension shows a Disable button (not Install)
  • If it shows Install, click it and wait for installation to finish
  • Reload VS Code after installing (close and reopen)
I installed Python but it still isn’t found when I run code in VS Code

VS Code caches the Python interpreter list at startup. If you installed Python after opening VS Code, it may not know about the new installation yet.

Fix:

  • Close VS Code completely and reopen it
  • Open a .py file and use Python: Select Interpreter from the Command Palette to pick the correct version
  • If no Python 3.10+ appears in the list, confirm the installation was successful by running python3 --version (Mac) or python --version (Windows) in a terminal outside VS Code
VS Code shows multiple Python versions — which one do I pick?

Multiple versions are common on Mac, where older versions may exist alongside newer ones. This is not a problem.

Fix: Pick the highest version that is 3.10 or above. Look for an entry like Python 3.12.x and select it. The version number shows in the VS Code bottom-right corner when a .py file is open. As long as it shows 3.10 or higher, you are set.

python --version shows Python 2.7 on Mac

On Mac, the python command (without the 3) sometimes still points to the old Python 2.7 that Apple bundled with older versions of macOS. Python 2 is end-of-life and is not used in this course.

Fix: Always use python3 on Mac, never plain python. Run:

python3 --version

This should return Python 3.10 or higher. If it does not, install Python 3 from python.org/downloads.

DataCamp isn’t showing up or I don’t have access

DataCamp access is granted through a course invite link distributed in Week 1. There is nothing to set up before then.

Fix: Wait for the invite email from the instructor, which arrives at your USF email during Week 1. Accept the invite and log in with your USF email address. If you accepted the invite but still do not have access, contact the instructor with your USF email and DataCamp username.

Common Python Errors
How to read an error message: Start at the bottom of the red output. The last line tells you the error type and message. The line above it tells you the file name and line number. That is where to look first.
SyntaxError: invalid syntax — Python won’t run my file at all

A SyntaxError means Python cannot even parse the file — there is a structural problem before any code runs. The error message includes a line number: go to that line first.

Most common causes:

  • Missing colon : at the end of an if, for, while, or def line
  • Unclosed parenthesis ( or bracket [ — often on the line before the reported line
  • Mismatched or unclosed quotation marks
  • Using = instead of == inside an if condition
# Wrong — missing colon
if x > 10
    print(x)

# Correct
if x > 10:
    print(x)
IndentationError — unexpected indent or expected an indented block

Python uses indentation (whitespace at the start of a line) to define code blocks. Every line inside an if, for, or def must be indented consistently.

Fix:

  • Use 4 spaces per indent level — never tabs mixed with spaces
  • In VS Code, go to View → Appearance → Render Whitespace to see exactly what is there
  • VS Code shows a Spaces / Tab Size indicator in the bottom-right corner; click it and choose Indent Using Spaces: 4
# Wrong — missing indent after def
def greet(name):
print("Hello", name)   # IndentationError

# Correct
def greet(name):
    print("Hello", name)
NameError: name 'x' is not defined

You used a variable before assigning it a value, or the name has a typo.

Common causes:

  • Spelling mistake: totla instead of total
  • Using a variable outside the function it was defined in (scope issue)
  • Running cells out of order in a Jupyter notebook
# Wrong — total not defined yet
print(total)
total = 0

# Correct
total = 0
print(total)
TypeError: can only concatenate str (not “int”) to str

You tried to combine a string and a number using +. Python does not do this automatically — you must convert the number to a string first.

Fix: Use an f-string (the preferred approach) or str():

# Wrong
age = 21
print("You are " + age + " years old")   # TypeError

# Option 1 — f-string (recommended)
print(f"You are {age} years old")

# Option 2 — str() conversion
print("You are " + str(age) + " years old")
TypeError: … object is not subscriptable or not iterable

“Not subscriptable” means you used [index] on something that is not a list or string — most commonly a number or None.

“Not iterable” means you tried to loop over something that is not a sequence — again, often a number or None.

Fix: Check what type the variable actually is. Add a print(type(variable)) line before the error to see what you are working with.

result = 42
print(result[0])   # TypeError: 'int' is not subscriptable

# Fix: result should be a list
result = [42]
print(result[0])   # 42
IndexError: list index out of range

You tried to access a list position that does not exist. If a list has 3 items, valid indexes are 0, 1, and 2. Accessing index 3 causes this error.

names = ["Ana", "Ben", "Cai"]
print(names[3])   # IndexError — only 0, 1, 2 exist

# Safe approach: check length first
if len(names) > 3:
    print(names[3])

Common cause in loops: using range(len(items) + 1) instead of range(len(items)).

KeyError: ‘key_name’ — dictionary lookup fails

You accessed a dictionary with a key that does not exist in it. Usually a typo or a key that only exists in some records.

record = {"name": "Ana", "score": 95}
print(record["scroe"])   # KeyError — typo, should be "score"

Fix:

  • Check the spelling of the key name
  • Use print(record.keys()) to see what keys are actually present
  • Use .get() to provide a default value instead of crashing: record.get("score", 0)
AttributeError: ‘NoneType’ object has no attribute ‘…’

A function returned None (nothing) instead of the value you expected. You then tried to call a method on that None value.

# Wrong — sort() modifies in place and returns None
names = ["Cai", "Ana", "Ben"]
sorted_names = names.sort()
print(sorted_names[0])   # AttributeError: NoneType

# Fix — use sorted() which returns a new list
sorted_names = sorted(names)
print(sorted_names[0])   # "Ana"

Fix: Check what your function actually returns. Add print(result) before the line that crashes to confirm it is not None.

FileNotFoundError: No such file or directory

Python could not find the file at the path you gave it. Either the file does not exist, the name is spelled wrong, or you are running the script from a different directory than you think.

Fix:

  • In the VS Code terminal, run pwd to see where Python is looking
  • Run ls to see what files are in that directory
  • Move the data file into the same folder as your script, then reference it by filename only: open("data.csv") not open("Downloads/data.csv")
ZeroDivisionError: division by zero

Your code attempted to divide a number by zero. This often happens when a variable that should contain a count is 0 because no data was processed yet, or because of a data quality issue.

# Fix: guard before dividing
if total > 0:
    average = revenue / total
else:
    average = 0
My if condition is never true even though it should be

The most common cause is using = (assignment) instead of == (comparison) in a condition. In Python 3 this is a SyntaxError inside an if, but it shows up as wrong logic in other situations.

The second most common cause: comparing a string to a number, or comparing with the wrong case.

# Wrong — comparing with wrong type
score = input("Enter score: ")  # input() returns a string
if score > 90:                  # TypeError or always False
    print("A")

# Fix — convert first
score = float(input("Enter score: "))
if score > 90:
    print("A")
My code runs but nothing prints — the terminal is blank

Two common causes:

  • Missing parentheses: In Python 3, print is a function and requires parentheses: print("hello") not print "hello"
  • Code inside a function that is never called: If your code is inside a def block but you never call the function, nothing runs
# Wrong — Python 3 requires parentheses
print "Hello"   # SyntaxError in Python 3

# Correct
print("Hello")

# Wrong — function defined but never called
def run():
    print("Hello")
# nothing printed!

# Correct
def run():
    print("Hello")
run()   # call it
