Why this matters

A variable is a container with a label on it. That is it. Once you have variables, types, and a way to print them, you have a tiny calculator that remembers what you told it. Every useful program ever written — from a simple report to a machine-learning pipeline — starts with variables that hold data. Understanding them is the foundation of all Python that follows.

The four data types introduced this module — str, int, float, and bool — map directly onto the kinds of information businesses track every day. A product name is a string. A price is a float. A quantity on hand is an integer. Whether a product is in stock is a boolean. When you look at a database table or a spreadsheet, every column is one of these types. Python's type system lets you express and manipulate that data with precision.

F-strings, introduced alongside print() this module, are how Python programs speak to humans. A script that computes revenue means nothing if it cannot display the result in a readable format. The ability to embed variables directly inside output strings — f"Revenue: ${revenue:.2f}" — is something you will use in every program you write this semester. Get comfortable with f-strings now and they will feel like second nature by midterm.

Core Concepts

Variable

A named reference to a value stored in memory. You create one with a single =: product_name = "Widget A". The name is on the left, the value on the right. The variable holds the value until you reassign it.

str (String)

Text enclosed in quotes. "Widget A", 'hello', "Tampa, FL" are all strings. Strings can be concatenated, sliced, and formatted. Use them for names, labels, descriptions, and any non-numeric data.

int (Integer)

A whole number with no decimal point: 42, -7, 1000. Use integers for counts, quantities, years, and anything you would never divide to get a fraction (except when you do, yielding a float).

float (Floating-point)

A number with a decimal point: 9.99, 0.07, -3.14. Use floats for prices, rates, percentages, and measurements. Be aware that floating-point arithmetic can introduce tiny rounding errors.

bool (Boolean)

Either True or False — no quotes, capitalized. Booleans are the result of comparison expressions and control flow decisions. in_stock = True is a natural business variable.

f-string

A string prefixed with f where expressions in {} are evaluated and inserted: f"Price: ${unit_price:.2f}". The :.2f format spec rounds to 2 decimal places. F-strings are the cleanest way to build output in Python.

Business Scenario

A merchandise planner at a retail chain needs a quick script to evaluate new product proposals. For each candidate product, she has a name, a unit cost from the supplier, a planned retail price, and an estimated quantity for the first order. She wants to print a one-line summary for each product showing the revenue potential and gross margin. Before this module she would set up an Excel template and type values into cells. After this module she writes a five-line Python script.

The script stores product_name = "Bamboo Water Bottle", unit_cost = 4.80, retail_price = 14.99, and quantity = 500. It computes revenue = retail_price * quantity and margin = (retail_price - unit_cost) / retail_price, then prints f"{product_name}: Revenue ${revenue:,.2f} | Margin {margin:.1%}". The output reads: Bamboo Water Bottle: Revenue $7,495.00 | Margin 67.9%. Clear, precise, reusable.

Key Vocabulary

  • Variable — a named location in memory that holds a value; assigned with =
  • Assignment — the act of binding a value to a variable name: x = 10
  • str — the Python type for text data; values are enclosed in single or double quotes
  • int — the Python type for whole numbers (no decimal)
  • float — the Python type for decimal numbers
  • bool — the Python type for True or False values
  • print() — a built-in function that writes output to the terminal
  • input() — a built-in function that pauses the program and reads a string the user types
  • f-string — a formatted string literal: f"Hello, {name}!"; variables in {} are substituted
  • type() — a built-in function that returns the data type of any value: type(3.14) returns <class 'float'>
Common Misconceptions
  • "True and 'True' are the same." They are not. True (no quotes) is a boolean. 'True' (with quotes) is a string containing the letters T-r-u-e. They behave completely differently in comparisons and conditions.
  • "= and == mean the same thing." Single = assigns a value to a variable. Double == compares two values and returns a boolean. Confusing them is one of the most common beginner bugs.
  • "input() returns a number if I type a number." input() always returns a string, regardless of what the user types. To use the input as a number, you must convert it: int(input("Qty: ")) or float(input("Price: ")).
