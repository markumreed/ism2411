Why this matters

Loops are where Python finally beats Excel. Excel can do a formula on one row, then you drag down. Python writes one loop and runs it on a million rows in a second. The pattern is always the same: initialize, loop, update. Once you internalize that pattern, you can process any collection of data — transactions, employees, products, log entries — with the same three-step mental model.

Consider what a financial analyst actually does every day: calculate the total revenue across all product lines, find the top-performing sales rep, flag any transactions over a threshold, compute a running average. Every one of those tasks is a loop. Before this module, you would need a spreadsheet or a pivot table to do it. After this module, you can do it in code — which means you can automate it, schedule it, and apply it to fresh data without touching it again.

There is also a conceptual shift that happens when you learn loops: you start thinking in terms of collections rather than individual values. A list of 10,000 sales is not 10,000 separate problems — it is one problem that runs 10,000 times. That shift in perspective is fundamental to writing real software, and it is what separates someone who "knows a bit of Python" from someone who can actually use it on the job.

Core Concepts

for loop

Iterates over every item in a sequence — a list, a string, or a range. Python pulls each item out one at a time, assigns it to your loop variable, and runs the body. Think of it as: "do this once for every item in the collection."

range()

Generates a sequence of integers. range(5) gives 0–4; range(2, 6) gives 2–5; range(0, 10, 2) gives 0, 2, 4, 6, 8. The stop value is always excluded — like a half-open interval in math.

while loop

Repeats as long as a condition is True. Use it when you do not know in advance how many iterations you need. The danger: if the condition never becomes False, the loop runs forever — an infinite loop that crashes your program.

break / continue

break exits the loop immediately — even if there are more items to process. continue skips the rest of the current iteration and moves to the next item. Both are used sparingly; overuse makes loops hard to follow.

Accumulator pattern

The most common loop pattern in data work: (1) initialize a variable before the loop (total = 0), (2) loop over the data, (3) update the variable each iteration (total += sale). Used for sums, counts, maximums, string building, and more.

Business Scenario

Your manager gives you a spreadsheet with 500 rows of daily sales transactions and asks three questions: What was the total revenue this week? What was the average transaction size? Which transaction was the largest? In Excel you could use SUM, AVERAGE, and MAX — but the boss also wants these numbers broken down by product category, and the next dataset will have 50,000 rows.

With Python, you write the logic once and it handles any size dataset. The code below demonstrates all three calculations using the accumulator pattern. Notice that we track three separate accumulators — total, count, and current_max — all updated inside the same loop in a single pass through the data.

sales = [120, 80, 250, 175, 90, 410, 60, 215]

total = 0           # accumulator 1: sum
count = 0           # accumulator 2: number of transactions
current_max = 0     # accumulator 3: largest sale

for sale in sales:
    total += sale
    count += 1
    if sale > current_max:
        current_max = sale

average = total / count

print(f"Total revenue:   ${total}")
print(f"Transaction count: {count}")
print(f"Average sale:    ${average:.2f}")
print(f"Largest sale:    ${current_max}")

Output: Total revenue: $1400 | Transaction count: 8 | Average sale: $175.00 | Largest sale: $410

Key Vocabulary

  • Iteration — the process of repeating a block of code for each item in a sequence
  • Iterable — any object you can iterate over: lists, strings, ranges, tuples, files, and more
  • Loop variable — the temporary variable (e.g., sale in for sale in sales:) that holds the current item
  • Accumulator — a variable initialized before the loop and updated inside it, collecting a result
  • Infinite loop — a while loop whose condition never becomes False; usually a bug caused by forgetting to update the loop variable
  • break — exits the loop immediately, skipping remaining iterations
  • continue — skips the rest of the current iteration and moves to the next one
  • range() — a built-in function that generates a sequence of integers; stop value is excluded
Common Misconceptions

1. "range(5) gives me 1 through 5." No — range(5) gives 0, 1, 2, 3, 4. The stop value is excluded. If you want 1 through 5, use range(1, 6).

2. "I should use a while loop when I'm not sure how many times to loop." Only partly true. If you are iterating over a collection (list, file, database results), use a for loop — even if you don't know the length. While loops are for situations where the stopping condition depends on something that changes during execution, not on exhausting a collection.

3. "break and continue are the same thing." Very different. break ends the entire loop. continue just skips to the next iteration — the loop keeps going.
