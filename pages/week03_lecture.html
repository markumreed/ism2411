Variables, Data Types & First Output

Now we actually write code. Three things today: variables, the four basic types, and how to print them out using f-strings.

Today's walkthrough

Variables are labeled containers

A variable is a name attached to a value. You assign with = (single equals). The name is on the left, the value is on the right. This is the single most important concept in all of programming: naming a piece of data so you can refer to it again. Without variables, every time you wanted to use a value, you would have to recompute it from scratch.

Variable names should be descriptive. p is technically valid but tells you nothing. unit_price tells you exactly what the variable contains. Python convention uses snake_case for variable names — all lowercase, words separated by underscores. Good names are the difference between code that is self-documenting and code that requires a comment every other line.

Variables can be reassigned. x = 5 followed by x = x + 1 gives x the value 6. The old value is gone. This is how you accumulate totals, update running counts, and build up results step by step in a program.

product = "Notebook"
price = 4.99
quantity = 12

The four types you will use most

Python has many types. These four cover almost everything you will do in this course. Every database column, every spreadsheet cell, every API response field maps onto one of these four types. When you read a CSV, every value arrives as a string; you must convert to int or float to do math with it. Knowing your types prevents an entire class of bugs before they happen.

Use type() to check what type a value has. This is invaluable when debugging. If your math is producing wrong results, print type(my_variable) and you will frequently discover it is a string when you expected a number. This is so common it has become reflex for experienced programmers: when something is wrong, check the types first.

name = "Alice"      # str  — text in quotes
age = 30             # int  — whole number
price = 19.99        # float — decimal number
in_stock = True      # bool — True or False (no quotes)

print(type(name))    # <class 'str'>
print(type(age))     # <class 'int'>
print(type(price))   # <class 'float'>
print(type(in_stock))# <class 'bool'>

print() and f-strings

print() writes to your terminal. An f-string is a string with an f in front — variables in curly braces get substituted in. F-strings were introduced in Python 3.6 and are now the standard way to format output in Python. They are more readable and faster than the older .format() method or string concatenation with +.

The format specification inside the curly braces gives you control over number formatting. {price:.2f} rounds to 2 decimal places. {revenue:,.0f} adds a thousands separator and rounds to a whole number. {margin:.1%} multiplies by 100 and adds a percent sign. These format specs come from Python's mini-language for formatting numbers — learning the basic ones pays off every time you print a report.

product = "Notebook"
price = 4.99
quantity = 12
total = price * quantity
print(f"{quantity} x {product} = ${total:.2f}")
# 12 x Notebook = $59.88

revenue = 150000
print(f"Revenue: ${revenue:,.0f}")
# Revenue: $150,000

margin = 0.342
print(f"Margin: {margin:.1%}")
# Margin: 34.2%

input() for user input

input() pauses the program and waits for the user to type something and press Enter. It always returns a string — convert with int() or float() if you need a number. This is a critical point: typing "42" into a terminal prompt does not give you the integer 42, it gives you the string "42". Attempting math with a string will raise a TypeError or produce unexpected results like string repetition.

The pattern int(input("How many? ")) combines the input() call with an immediate conversion. The inner function runs first (reads a string from the user), and the outer function runs second (converts that string to an integer). This nested-function pattern is very common in Python and worth getting comfortable with.

name = input("Your name: ")
print(f"Hello, {name}!")

# Convert input to a number:
quantity = int(input("How many units? "))
price = float(input("Unit price: $"))
total = quantity * price
print(f"Total: ${total:.2f}")

Jupyter vs. VS Code

A .py file in VS Code is one big script that runs top to bottom. A Jupyter notebook is a series of cells you can run one at a time, with output (including charts) right next to the code. We use VS Code for scripts and Jupyter for exploration — you will see both this semester. Jupyter is especially useful for data analysis because you can run a cell, inspect the result, and decide what to do next without re-running the entire script from the top.

Jupyter's cell-based model also means variables persist across cells within the same notebook session. If you run a cell that assigns x = 10, then run a different cell that prints x, you will see 10. This is powerful for exploration but can cause confusion: if you re-run an earlier cell, it may overwrite a variable set by a later cell. The rule of thumb is: when your notebook's results seem wrong, restart the kernel and run all cells in order to get a clean state.

Detailed Code Walkthrough

A complete product summary script

This script brings together variables, types, f-strings, and simple arithmetic into a useful tool. Read each line and make sure you can predict the output before running it.

# product_summary.py
# Stores product data in variables and prints a formatted summary

product_name = "Bamboo Desk Organizer"   # str
unit_cost    = 8.50                       # float — what we paid the supplier
retail_price = 24.99                      # float — what we charge the customer
quantity     = 200                        # int   — units in inventory
in_stock     = True                       # bool

# Compute derived values
revenue      = retail_price * quantity    # 200 units sold at retail
margin       = (retail_price - unit_cost) / retail_price

# Print the report
print(f"Product:   {product_name}")
print(f"Price:     ${retail_price:.2f}")
print(f"Quantity:  {quantity:,}")         # :, adds thousands separator
print(f"Revenue:   ${revenue:,.2f}")
print(f"Margin:    {margin:.1%}")
print(f"In Stock:  {in_stock}")

The interactive version with input()

The same logic but now the user supplies the data at runtime. Notice the conversions on lines 3–5.

# pricer_interactive.py
product_name = input("Product name: ")
retail_price = float(input("Retail price: $"))
unit_cost    = float(input("Unit cost:    $"))
quantity     = int(input("Quantity:     "))

revenue = retail_price * quantity
margin  = (retail_price - unit_cost) / retail_price

print()  # blank line for readability
print(f"--- {product_name} Summary ---")
print(f"Revenue: ${revenue:,.2f}   Margin: {margin:.1%}")

Common Errors

Forgetting to convert input() to a number

# WRONG — user types "10" but we get the string "10"
quantity = input("Quantity: ")
total = quantity * 4.99   # TypeError: can't multiply str by float

# CORRECT — convert before doing math
quantity = int(input("Quantity: "))
total = quantity * 4.99   # works: 49.9

Confusing = (assignment) and == (comparison)

# WRONG — this assigns 10 to x and always evaluates True
if x = 10:   # SyntaxError in Python — Python is strict about this

# CORRECT — use == to compare
if x == 10:
    print("x is ten")

Printing a variable name instead of its value

# WRONG — prints the literal text "price", not the value
print("price")    # output: price

# CORRECT — no quotes around a variable name, or use f-string
print(price)               # output: 19.99
print(f"Price: {price}")   # output: Price: 19.99

Key Takeaways
  • A variable is a name bound to a value in memory; use = to assign.
  • The four core types are str, int, float, and bool; use type() to inspect.
  • F-strings (f"...") let you embed variables and expressions directly in output strings.
  • input() always returns a string — convert with int() or float() before doing math.
  • Use VS Code for scripts that run top to bottom; use Jupyter for exploratory, cell-by-cell analysis.
