Today's walkthrough

groupby — split, apply, combine

The groupby() method implements the "split-apply-combine" pattern: pandas splits the DataFrame into groups based on a column's unique values, applies an aggregation function (sum, mean, count, etc.) to each group independently, then combines the results into a single Series or DataFrame. This mirrors the behavior of a SQL GROUP BY clause or an Excel PivotTable — but it is faster, reproducible, and composable.

The most common pattern is df.groupby("column")["value_col"].function(). The result is a Series indexed by the group labels. Call .reset_index() to convert it into a two-column DataFrame that you can pass directly to a plotting function.

df.groupby("region")["revenue"].sum()
df.groupby("region")["revenue"].mean()
df.groupby(["region","category"])["revenue"].sum()

# Multiple aggregations in one call:
df.groupby("region").agg({"revenue": "sum", "order_id": "count"}).reset_index()

Calculated columns

Create new columns before grouping to enable richer aggregations. The most important derived columns for business analytics are profit (revenue minus cost), margin (profit divided by revenue), and date parts (month, year, quarter, day of week). Date parts are extracted from a datetime column using the .dt accessor.

df["profit"] = df["revenue"] - df["cost"]
df["margin"] = df["profit"] / df["revenue"]
df["month"] = df["order_date"].dt.month
df["year"]  = df["order_date"].dt.year
df["quarter"] = df["order_date"].dt.quarter

A bar chart with matplotlib

matplotlib is Python's foundational plotting library. It gives you complete control over every visual element. The standard workflow is: create a figure, call the chart function, add labels, and either save or show. Always set figsize so the chart is not too small when saved. Always add a title and axis labels — a chart without labels is uninterpretable to anyone who hasn't seen your data.

import matplotlib.pyplot as plt

region_revenue = df.groupby("region")["revenue"].sum().reset_index()
plt.figure(figsize=(10, 5))
plt.bar(region_revenue["region"], region_revenue["revenue"], color="#fbbf24")
plt.title("Revenue by Region", fontsize=14, fontweight="bold")
plt.ylabel("Revenue ($)")
plt.xlabel("Region")
plt.tight_layout()
plt.savefig("charts/revenue_by_region.png", dpi=150)
plt.show()

A line chart for trends over time

Line charts communicate that time is on the x-axis and magnitude is on the y-axis, and that consecutive points are connected. Use them for monthly revenue, weekly signups, or quarterly growth — anything where the relationship between adjacent periods is meaningful. Add markers to make individual data points visible, especially when there are fewer than 20 data points.

monthly = df.groupby("month")["revenue"].sum().reset_index()
plt.figure(figsize=(10, 5))
plt.plot(monthly["month"], monthly["revenue"], marker="o", color="#60a5fa", linewidth=2)
plt.title("Monthly Revenue Trend")
plt.xlabel("Month")
plt.ylabel("Revenue ($)")
plt.xticks(range(1, 13))
plt.savefig("charts/monthly_trend.png", dpi=150)
plt.show()

A histogram for distributions

Histograms divide a continuous variable's range into equal-width "bins" and show how many observations fall into each bin. They answer questions like: "Are most orders small with a few giant ones?" (right-skewed) or "Do orders cluster around a typical size?" (bell-shaped). The number of bins is a judgment call — too few loses detail, too many shows noise. Start with 20 and adjust.

plt.figure(figsize=(8, 5))
plt.hist(df["revenue"], bins=20, color="#34d399", edgecolor="black", alpha=0.8)
plt.title("Order Size Distribution")
plt.xlabel("Revenue ($)")
plt.ylabel("Number of Orders")
plt.savefig("charts/order_distribution.png", dpi=150)
plt.show()

Seaborn for quick stats charts

Seaborn sits on top of matplotlib and gives you cleaner default styles, automatic aggregation, and stats-aware additions like confidence intervals and regression lines. Its API is DataFrame-native: you pass the entire DataFrame and specify column names as strings. For the capstone, Seaborn is an excellent choice for category-level charts and distribution plots — it takes fewer lines and looks more professional out of the box.

import seaborn as sns
sns.set_theme(style="darkgrid")

# Bar chart with confidence intervals (automatic)
sns.barplot(data=df, x="region", y="revenue", ci=95)

# Box plot: shows median, quartiles, and outliers per group
sns.boxplot(data=df, x="region", y="revenue")

# Histogram (Seaborn version)
sns.histplot(df["revenue"], bins=20, kde=True)

Choosing the right chart

Chart type selection is a communication decision, not a cosmetic one. Bar = compare discrete categories. Line = trend over time (sequential order matters). Histogram = distribution of one continuous variable. Scatter = relationship between two continuous variables. Box plot = distribution across groups with outlier visibility. Heatmap = magnitude across two categorical axes. Never use a pie chart for more than 4 slices — use a bar chart instead, always.

Live Coding Walkthrough

import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

df = pd.read_csv("data/clean_sales.csv")
df["order_date"] = pd.to_datetime(df["order_date"])
df["month"] = df["order_date"].dt.month
df["revenue"] = df["unit_price"] * df["quantity_sold"]

# ── Q1: Which region drives the most revenue? ─────────────────
by_region = df.groupby("region")["revenue"].sum().reset_index()
by_region = by_region.sort_values("revenue", ascending=False)

plt.figure(figsize=(8,5))
plt.bar(by_region["region"], by_region["revenue"], color="#fbbf24")
plt.title("Total Revenue by Region")
plt.ylabel("Revenue ($)")
plt.tight_layout()
plt.savefig("charts/q1_region.png", dpi=150)
plt.show()
print("Top region:", by_region.iloc[0]["region"],
      f"${by_region.iloc[0]['revenue']:,.0f}")

# ── Q2: How does revenue trend by month? ──────────────────────
monthly = df.groupby("month")["revenue"].sum().reset_index()

plt.figure(figsize=(10,5))
plt.plot(monthly["month"], monthly["revenue"], marker="o", linewidth=2, color="#60a5fa")
plt.title("Monthly Revenue Trend")
plt.xlabel("Month"); plt.ylabel("Revenue ($)")
plt.xticks(range(1,13))
plt.tight_layout()
plt.savefig("charts/q2_monthly.png", dpi=150)
plt.show()

# ── Q3: What does the order size distribution look like? ──────
plt.figure(figsize=(8,5))
sns.histplot(df["revenue"], bins=30, kde=True, color="#34d399")
plt.title("Order Size Distribution")
plt.xlabel("Revenue ($)"); plt.ylabel("Count")
plt.tight_layout()
plt.savefig("charts/q3_distribution.png", dpi=150)
plt.show()

print(f"Median order: ${df['revenue'].median():,.2f}")
print(f"Mean order:   ${df['revenue'].mean():,.2f}")

Common Errors

Error 1: savefig after show saves blank file

# WRONG
plt.hist(df["revenue"])
plt.show()         # clears the figure
plt.savefig("x.png")  # saves blank!

# RIGHT
plt.hist(df["revenue"])
plt.savefig("x.png")  # save BEFORE show
plt.show()

Error 2: groupby returns a Series, bar() needs two arrays

# WRONG — result is a Series, not two separate arrays
result = df.groupby("region")["revenue"].sum()
plt.bar(result)  # TypeError

# RIGHT — use reset_index() then index into both columns
result = df.groupby("region")["revenue"].sum().reset_index()
plt.bar(result["region"], result["revenue"])

Error 3: Forgetting plt.tight_layout() causes cut-off labels

# Long category names get cut off without tight_layout
plt.bar(categories, values)
plt.xticks(rotation=30, ha="right")
plt.tight_layout()  # prevents label clipping
plt.savefig("chart.png", bbox_inches="tight")  # extra safety margin

Key Takeaways
  • groupby() is the primary tool for aggregate business questions — split by category, apply a function, combine results.
  • Always call .reset_index() after groupby when you need a plain DataFrame for plotting or further analysis.
  • Chart labeling is non-negotiable: title, x-axis label, y-axis label — every chart, every time.
  • Call plt.savefig() before plt.show() — show() clears the figure.
  • Chart type choice communicates your analytical judgment: bar for categories, line for time, histogram for distributions.
