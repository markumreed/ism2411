Product Pricer with f-strings

Write a small program that combines variables, types, and f-strings into a useful tiny tool.

Module 03 · Lab Unit 1
This module: Reading Lecture Lab

Exercises

EXERCISE 1 BASIC
Variables and types
Create variables for product (str), unit_price (float), quantity (int), in_stock (bool). Print each one with print(type(...)) to confirm types.
Expected: four lines like <class 'str'>, <class 'float'>, <class 'int'>, <class 'bool'>
EXERCISE 2 BASIC
f-string practice
Print a one-line summary like: "12 units of Notebook at $4.99 each = $59.88". Compute the total inside the f-string with {unit_price * quantity}. Use the :.2f format spec on the total.
Expected: 12 units of Notebook at $4.99 each = $59.88
EXERCISE 3 BASIC
Input version
Modify the script to ask the user for product, price, and quantity via input(). Convert price to float and quantity to int. Print the same formatted summary line.
Expected: program prompts for three values, then prints the formatted summary using the user-supplied data
EXERCISE 4 INTERMEDIATE
Multi-variable product card
Add a unit_cost variable. Compute and print a five-line product card:
Product: Notebook
Price: $4.99
Qty: 12
Revenue: $59.88
Margin: 34.1%
Use f-strings with format specs for all numeric output.
Expected: five-line formatted card as shown above (with your own product values)
EXERCISE 5 INTERMEDIATE
Type investigation
Open the Python interactive interpreter (type python3 in terminal, no filename). Try: type("12"), type(12), type(12.0), type(True). Then try: "5" + "3" vs. 5 + 3. Write down what you observe about how Python handles these differently.
Expected: written observations — "5" + "3" produces "53" (concatenation); 5 + 3 produces 8 (addition)
EXERCISE 6 INTERMEDIATE
Stretch — Jupyter
Open a Jupyter notebook (jupyter notebook in the terminal, or use VS Code's built-in notebook support). Recreate the product card script across four cells: Cell 1 — assign variables; Cell 2 — compute derived values; Cell 3 — print summary; Cell 4 — a markdown cell explaining what the script does. Run them in order, then try running Cell 3 before Cell 1 and observe what happens.
Expected: cells 1-3 produce the product card; running Cell 3 first raises NameError because variables are not yet defined

Stretch Challenge

STRETCH A CHALLENGE
Three-product comparison
Extend your script to handle three products. Use distinct variable names for each (e.g., product1, price1, cost1, etc.). Print a formatted product card for each and a final line showing which product has the highest margin. Use comparison operators to determine the winner.
STRETCH B CHALLENGE
Format spec exploration
In the Python interactive interpreter, try each of these format specs and note what they do: f"{1234567:.2f}", f"{1234567:,.0f}", f"{0.3456:.1%}", f"{42:05d}", f"{'hello':>20}". Write a one-sentence description of each. This is the Python format mini-language — useful for professional-quality output.

Reflection

Self-Assessment Questions
  1. What happened when you ran Exercise 3 and forgot to convert the price from string to float? Describe the exact error message and explain in your own words why it occurred.
  2. In Exercise 5, "5" + "3" returned "53" instead of 8. Why does Python do this, and how does this explain why input() must always be converted before doing math?
  3. How would you use variables, types, and f-strings to build a real tool for a task in your future career? Describe a specific scenario in 2–3 sentences.
Submission

Submit pricer.py to Canvas. Filename matters — exactly pricer.py. Your file should include exercises 1–4 as a single runnable script with comments separating each exercise.
