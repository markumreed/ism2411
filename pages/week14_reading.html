Why this matters

There is a widely quoted estimate among data professionals: roughly 80% of analytical work is data cleaning. That number feels dramatic until the first time you try to compute an average on a column where some values are stored as "$1,200.00" — a string with a dollar sign, a comma, and quotes. pandas will happily load that column, but every arithmetic operation on it will fail with a TypeError until you fix the type. Module 14 is about developing the reflexes to spot these problems before they sink your analysis.

Real business data comes from many sources: CRM systems export dates in inconsistent formats, ERP downloads include header rows mixed with data, manual data entry produces "N/A", "n/a", "NA", and empty cells all meaning the same thing. A data analyst who cannot handle these variations cannot deliver results that anyone trusts. Learning to clean data is not a secondary skill — it is often the primary differentiator between an analyst who ships and one who gets stuck.

The descriptive statistics side of this module is equally practical. Before you answer any business question, you need to understand the data's basic shape: how many rows, how spread out are the values, what is the range, are there impossible values (negative quantities, prices of zero for a premium product)? Running df.describe() in the first 60 seconds of any new dataset is a professional habit worth building now. It takes five seconds and has saved countless hours of downstream confusion.

Core Concepts

isnull() / isna()

Returns a boolean DataFrame where True = missing. Chain with .sum() to count missing values per column. df.isnull().sum() is the first diagnostic you run on any new dataset.

dropna()

Remove rows with missing values. Use subset=["col"] to drop only when a specific column is missing. Default drops rows with ANY missing value — often too aggressive for real data.

fillna()

Replace missing values with a constant or computed value. df["price"].fillna(0) fills with zero; df["price"].fillna(df["price"].mean()) fills with the column mean — a common statistical imputation technique.

astype() / pd.to_datetime()

df["price"].astype(float) converts a column's dtype. pd.to_datetime(df["date"]) parses date strings into proper datetime objects so you can extract month, year, day-of-week, etc.

describe()

Returns count, mean, std, min, 25th/50th/75th percentile, and max for all numeric columns in one call. Use it to spot impossible values: negative quantities, prices of exactly zero, max values that are 100× the mean (outliers).

Business Scenario

Your company's sales team exports monthly data from a legacy order management system. The export has 4,800 rows and the following problems: the Price column stores values as strings like "$1,200.00"; the Order Date column has entries in three different date formats (2024-01-15, 01/15/2024, and January 15 2024); 7% of rows are missing a Customer ID; and column names have spaces and capital letters. Here is the cleaning script you would write:

import pandas as pd

df = pd.read_csv("data/messy_sales.csv")

# Step 1: Standardize column names
df.columns = df.columns.str.lower().str.replace(" ", "_")
# "Order Date" → "order_date", "Customer ID" → "customer_id"

# Step 2: Count missing values — always do this first
print(df.isnull().sum())

# Step 3: Drop rows missing essential identifiers
df = df.dropna(subset=["customer_id"])  # drop if customer_id is null

# Step 4: Fill numeric columns with column mean
df["quantity"] = df["quantity"].fillna(df["quantity"].mean())

# Step 5: Fix price — strip "$" and "," then convert to float
df["price"] = df["price"].str.replace("$", "", regex=False)
df["price"] = df["price"].str.replace(",", "", regex=False)
df["price"] = df["price"].astype(float)

# Step 6: Parse dates — pandas handles mixed formats with errors='coerce'
df["order_date"] = pd.to_datetime(df["order_date"], errors="coerce")
print(f"Bad dates: {df['order_date'].isnull().sum()}")  # how many failed?

# Step 7: Check for duplicates
print(f"Duplicate rows: {df.duplicated().sum()}")
df = df.drop_duplicates()

# Step 8: Descriptive stats on the clean data
print(df.describe())

# Step 9: Save clean data
df.to_csv("data/clean_sales.csv", index=False)
print("Saved clean_sales.csv")

Key Methods Reference

  • df.isnull().sum() — count missing values per column
  • df.info() — dtypes plus non-null count per column in one view
  • df.dropna() — drop rows with any missing value
  • df.dropna(subset=["col"]) — drop only if specific column is missing
  • df["col"].fillna(value) — replace NaN with value or expression
  • df.columns.str.lower().str.replace(" ","_") — standardize column names
  • df.rename(columns={"old": "new"}) — rename specific columns
  • df["col"].astype(float) — convert column to numeric type
  • pd.to_datetime(df["col"]) — parse date strings into datetime
  • df.duplicated().sum() — count duplicate rows
  • df.drop_duplicates() — remove duplicate rows
  • df.describe() — summary statistics for all numeric columns
  • df["col"].mean(), .median(), .mode(), .std() — individual stats
Common Misconceptions

Pitfall 1 — dropna() removes too many rows: By default, df.dropna() drops every row with even one missing value. On a dataset where 10 different columns each have 5% missing, you could lose 40% of your data. Use subset= to target only the columns that actually matter, and use fillna() for the rest.

Pitfall 2 — Forgetting to assign after cleaning: df["price"].fillna(0) does not modify df — it returns a new Series. You must write df["price"] = df["price"].fillna(0). The same applies to dropna(), drop_duplicates(), and most other pandas methods.

Pitfall 3 — Treating the mean as always the right fill value: Filling missing prices with the column mean sounds reasonable but can introduce bias if prices vary by category. Always think about whether the fill value makes business sense before applying it.
