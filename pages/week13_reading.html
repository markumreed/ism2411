Why this matters

pandas is the most-used data tool in business analytics, and for good reason: it collapses weeks of spreadsheet work into a few lines of code. A regional sales manager at a mid-size retailer recently described their old process — downloading a CSV, opening Excel, manually filtering by region, copy-pasting into a new tab, sorting by revenue — as "Tuesday mornings for two years." With pandas, that same operation is one chained expression that runs in under a second and produces an identical result every time.

Everything we built in Units 1–3 is still true and still relevant. Lists, loops, functions, dictionaries — pandas DataFrames are built on those same ideas, but wrapped in a layer designed specifically for tabular data. A DataFrame is, at its core, a dictionary of lists where each key is a column name and each value is a column of uniform data. Once you understand that, the rest follows naturally.

From this module forward, every dataset we touch goes through a DataFrame. The skills you build here — loading, viewing, filtering, and sorting — are the foundation for Module 14 (cleaning) and Module 15 (aggregation and visualization), which together make up your capstone toolkit. Analysts who can confidently navigate pandas are, according to multiple LinkedIn job-market analyses, more likely to receive callbacks for data-adjacent business roles than candidates who list only Excel.

Core Concepts

pd.read_csv()

Loads a CSV file into a DataFrame in one line. Handles headers, data types, and encoding automatically. Pass index_col=0 if your CSV has a row-number column.

DataFrame

A 2D table with labeled columns and a row index. Think of it as a spreadsheet in memory. Each column is a Series. Access columns with df["col"] or df[["a","b"]].

Series

A single column — a 1D array with an index. df["revenue"] returns a Series. You can do math on a whole Series at once: df["revenue"] * 1.1 raises every value by 10%.

Boolean filtering

Pass a condition inside brackets to keep only matching rows. df[df["region"] == "South"] returns every row where region is South. Use & (and) and | (or) for compound conditions — never the Python keywords.

sort_values()

Sort rows by any column. ascending=False for descending order. Chain with .head(10) to get top-N results. The original DataFrame is not modified unless you pass inplace=True.

Business Scenario

You are a business analyst at a regional retailer. Your manager emails you a CSV file called retail_sales.csv and asks: "Who are our top 10 customers by revenue this quarter, and are any of them in the South region?" In Excel this would take 15 minutes of filtering, sorting, and manual scanning. In pandas, you write the following and get the answer in seconds — with a reproducible script you can run again next quarter with zero changes.

import pandas as pd

# Load the 5,000-row dataset
df = pd.read_csv("data/retail_sales.csv")

# Quick inspection
print(df.shape)          # (5000, 6)
print(df.columns.tolist())
print(df.head())

# Top 10 orders by revenue
top10 = df.sort_values("revenue", ascending=False).head(10)
print(top10[["customer_id", "region", "revenue"]])

# Are any top-10 orders from the South?
south_top10 = top10[top10["region"] == "South"]
print(f"{len(south_top10)} of the top 10 orders are from the South region.")

Key Methods Reference

  • pd.read_csv("file.csv") — load a CSV file into a DataFrame
  • df.head(n) — first n rows (default 5)
  • df.tail(n) — last n rows (default 5)
  • df.info() — column names, non-null counts, data types
  • df.shape — tuple of (rows, columns)
  • df.columns — Index of column names
  • df.dtypes — data type of each column
  • df["col"] — select one column as a Series
  • df[["col1","col2"]] — select multiple columns as a DataFrame
  • df[df["col"] == value] — filter rows by condition
  • df[(cond1) & (cond2)] — filter rows by two conditions (use &, not and)
  • df.sort_values("col", ascending=False) — sort by column
  • df.to_csv("out.csv", index=False) — save DataFrame to CSV
Common Misconceptions

Pitfall 1 — Using and instead of &: In pandas, boolean filters must use & (and) and | (or), not Python's and/or keywords. df[(df["region"]=="South") and (df["revenue"]>1000)] raises a ValueError. Always wrap each condition in parentheses and use &.

Pitfall 2 — Confusing Series and DataFrame selection: df["revenue"] returns a Series (1D). df[["revenue"]] with double brackets returns a single-column DataFrame (2D). Some methods require one type; using the wrong one produces confusing errors.

Pitfall 3 — Forgetting that most methods return new objects: df.sort_values("revenue") does NOT sort df in place — it returns a new sorted DataFrame. You must assign the result: df = df.sort_values("revenue") or use inplace=True.

Q1 · Code Trace
What is the output? 
import pandas as pd
df = pd.DataFrame({'item':['pen','book'],'price':[1.5,12.0]})
print(df['price'].mean())
Answer
6.75 — (1.5+12.0)/2 = 6.75

  • b) Column names only

Q4 · Code Trace
What does this code return? 
df = df[df['region'] == 'South']

A DataFrame containing only rows where region equals 'South'
Q5 · Multiple Choice
What does df.shape return for a dataset with 5,000 rows and 6 columns?
  • a) 30000
  • b) (6, 5000)
  • c) (5000, 6)
  • d) [5000, 6]
Answer
(5000, 6) — shape always returns (rows, columns) as a tuple
Q6 · Code Trace
What does df.dropna() do? 
df = df.dropna()
Answer
Returns a new DataFrame with all rows containing at least one missing value removed
Q7 · Multiple Choice
What is the correct way to filter rows where revenue > 1000 AND region is "South"?
  • a) df[df["revenue"] > 1000 and df["region"] == "South"]
  • b) df[(df["revenue"] > 1000) & (df["region"] == "South")]
  • c) df[df["revenue"] > 1000, df["region"] == "South"]
  • d) df.filter(revenue=1000, region="South")
Answer
b) — use & (not and) and wrap each condition in parentheses
Q8 · Code Trace
What does this code produce? 
df['per_unit'] = df['revenue'] / df['quantity']
print(df['per_unit'].head(3))
Answer
A new column 'per_unit' is added to df (revenue divided by quantity for each row), then the first 3 values of that column are printed as a Series
Q9 · Multiple Choice
Which of the following returns a DataFrame (not a Series)?
  • a) df["revenue"]
  • b) df[["revenue"]]
  • c) df.revenue
  • d) df["revenue"].values
Answer
b) df[["revenue"]] — double brackets always return a DataFrame; single brackets return a Series
