Why this matters

A list says 'here are five products in order'. A dictionary says 'here is one product, and here is everything I know about it'. Dictionaries are the bridge to working with structured records — and a stepping stone to pandas DataFrames in Unit 4.

What you will learn this week

  • Key-value pairs
  • Nesting, adding, updating, and deleting entries
  • Iterating over dictionaries
  • Business context: customer records and lookup tables

Q1 · Code Trace
What is the output?
customer = {"name": "Alice", "age": 30}
print(customer["name"])
Answer
Alice
Q2 · Multiple Choice
How do you add a key-value pair to a dictionary?
  • a) dict.add("key", val)
  • b) dict["key"] = val
  • c) dict.insert("key", val)
  • d) dict.append("key", val)
Answer
dict["key"] = val
Q3 · Code Trace
What is the output?
product = {"name": "Laptop", "price": 999}
product["price"] = 849
print(product["price"])
Answer
849
Q4 · Multiple Choice
What does customer.get('email', 'Not found') return if 'email' is not in the dict?
  • a) None
  • b) An error
  • c) 'Not found'
  • d) 'email'
Answer
'Not found'
Q5 · Code Trace
What does this print?
inventory = {"apples": 10, "bananas": 5}
for item in inventory:
    print(item)
Answer
apples
bananas
— Iterating a dict returns its keys
Q6 · Multiple Choice
Which method returns all key-value pairs as tuples?
  • a) .keys()
  • b) .values()
  • c) .items()
  • d) .pairs()
Answer
.items()
