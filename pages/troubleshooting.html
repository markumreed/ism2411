Troubleshooting Guide

The most common errors in ISM2411, organized by type. Find your error, read the cause, apply the fix.

The one rule for reading any error

Read the traceback from the bottom up. The bottom line gives you the error type and the exact message. The line above gives you the file name and line number. Start there. Do not read from the top.

Environment & Files

ModuleNotFoundErrorFirst seen: Module 13
ModuleNotFoundError: No module named 'pandas'

Cause: The package is not installed for the Python that ran your code — often VS Code or Jupyter is using a different interpreter than the one you installed into.

Fix: Run pip install pandas (or pip3 install pandas). In VS Code, check the interpreter shown in the status bar; in Jupyter, check the kernel menu. Install into the one you are actually running.

FileNotFoundErrorFirst seen: Module 12
FileNotFoundError: [Errno 2] No such file or directory: 'data.csv'

Cause: Python could not find the file at the path you gave. Usually the file is fine — your working directory is not the folder you think it is.

Fix: Add import os; print(os.getcwd()) to see where Python is actually looking. Run the script from the folder that contains the file, or use the full path.

not a git repositoryFirst seen: Module 8
fatal: not a git repository (or any of the parent directories): .git

Cause: You ran a git command outside your project folder, or never ran git init in it.

Fix: cd into the project folder first. If it has never been initialized, run git init once.

Python Syntax & Logic

SyntaxErrorFirst seen: Module 5
SyntaxError: expected ':'

Cause: Python could not parse the file — a missing colon after if/def/for, an unclosed parenthesis or bracket, or a mismatched quote.

Fix: Go to the reported line, then check the line above it too — the actual mistake is often one line earlier than where Python noticed.

IndentationErrorFirst seen: Module 5
IndentationError: unexpected indent

Cause: A line is indented inconsistently — wrong number of spaces, or tabs mixed with spaces.

Fix: Use 4 spaces per level, never tabs. In VS Code, enable View → Render Whitespace to see exactly what is there.

NameErrorFirst seen: Module 7
NameError: name 'result' is not defined

Cause: A variable was used before it was assigned, its name is misspelled, or it was created inside a function (local scope) and accessed outside.

Fix: Check spelling first. If the variable lives inside a function, return it and capture the result instead of reaching for it directly.

TypeErrorFirst seen: Module 3
TypeError: can only concatenate str (not "int") to str

Cause: An operation was applied to the wrong type. The classic: input() always returns a string, and you did math with it.

Fix: Convert immediately: qty = int(input("Quantity: ")). For printing, use an f-string: f"Items: {count}".

ValueErrorFirst seen: Module 4
ValueError: could not convert string to float: '$24.99'

Cause: The type of the operation is right, but the content is not valid — converting a string that does not look like a number.

Fix: Clean the string first: float("$24.99".replace("$", "")). For whole numbers in a decimal string, use int(float("24.99")).

Data Structures & pandas

IndexErrorFirst seen: Module 10
IndexError: list index out of range

Cause: You asked for a position that does not exist. Lists are zero-indexed — a 3-item list has indexes 0, 1, and 2.

Fix: Print len(items) before indexing. For the last item, use items[-1].

KeyErrorFirst seen: Module 11
KeyError: 'amout' ← typo, should be 'amount'

Cause: A dictionary key (or pandas column name) does not exist — usually a typo, wrong capitalization, or an invisible trailing space in a CSV header.

Fix: Print record.keys() or df.columns and compare character by character. Use .get('key', default) for optional keys.

AttributeErrorFirst seen: Module 13
AttributeError: 'DataFrame' object has no attribute 'groupyb'

Cause: A misspelled method or attribute name — pandas method typos are the classic case.

Fix: Check the spelling against the cheat sheet (groupby, describe, to_csv). If unsure what an object offers, print(dir(df)).

Still stuck?

✓ Step 0 — explain it to the duck

Before asking anyone, explain the failing code out loud, one line at a time, to an inanimate object — rubber duck debugging, named in The Pragmatic Programmer (Hunt & Thomas, Addison-Wesley, 1999). Stating what each line literally does, instead of what you assumed it does, surfaces a surprising number of bugs mid-sentence. Full technique in the Module 7 reading.

If the duck doesn't crack it and the error persists:

  1. Copy the full traceback — all lines from "Traceback" to the last line, not just the last line
  2. Write one sentence describing what you were trying to do when it appeared
  3. Email your professor or TA using the format on the FAQ page, bring it to office hours, or post it to the class discussion board with your code pasted as text (not a screenshot)

Don't sit stuck for more than 20 minutes — the duck costs two of them, and the other eighteen are better spent with help on the way.
