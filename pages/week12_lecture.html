Today's walkthrough

Opening a file with with

The with block opens the file, runs your code, and closes the file automatically — even if an error occurs inside the block. This is critical: an unclosed file can leave data unwritten, lock the file on some operating systems, or exhaust the system's file handle limit in a long-running script.

Before the with statement was common practice, you had to call f.close() manually and wrap everything in a try/finally block to ensure cleanup. The with statement does all of that for you in one clean syntax.

The variable after as (conventionally named f) is the file object. You call methods on it — f.read(), f.readlines(), f.write() — inside the indented block. Once the block exits, the file is closed automatically.

with open("data.txt", "r") as f:
    contents = f.read()       # read entire file as one string
print(contents)               # file is already closed here

# Reading line by line (better for large files)
with open("data.txt", "r") as f:
    for line in f:
        print(line.strip())   # .strip() removes the newline character

Reading mode vs. writing mode

"r" reads. "w" writes — and immediately empties the file when opened, before you've written anything. This surprises many students and has caused real data loss. If you want to add to an existing file without clearing it, use "a" (append mode). If you need to read and write the same file, use "r+".

The mode string can also specify encoding: open("file.txt", "r", encoding="utf-8"). On Windows, the default encoding may not be UTF-8, which can cause strange characters when reading files that contain accented letters or special symbols. Always specify encoding="utf-8" for files you'll share across platforms.

open("file.csv", "r")                        # read only
open("file.csv", "w")                        # write — clears existing!
open("file.csv", "a")                        # append to end
open("file.csv", "r", encoding="utf-8")      # read with explicit encoding

Reading a CSV with the csv module

Don't parse commas by hand — the csv module handles quoted strings, escaped commas, and all the edge cases. A cell that contains a comma (like an address: "123 Main St, Suite 4") would break a naive line.split(",") approach. The csv module reads the file format correctly per the CSV specification.

csv.reader(f) returns an iterator — each iteration gives you one row as a list of strings. All values are strings regardless of what they look like; you must convert to int or float yourself when doing arithmetic. This is the most common csv bug: multiplying two strings gives string repetition, not a number.

import csv

with open("sales.csv", "r") as f:
    reader = csv.reader(f)
    next(reader)  # skip the header row
    for row in reader:
        product, price, qty = row
        # Convert types — csv.reader returns everything as strings
        revenue = float(price) * int(qty)
        print(f"{product}: ${revenue:.2f}")

Writing a CSV

Always pass newline="" as the third argument to open() when writing CSVs. Without it, on Windows, Python adds an extra \r to each line ending, producing blank rows between every data row in Excel. This is a platform-specific quirk that the Python docs explicitly document — use newline="" and let the csv module control line endings.

writer.writerow([...]) writes one row. writer.writerows([[...], [...]]) writes multiple rows at once. Always write the header row first.

import csv

rows = [["Alice", 95], ["Bob", 87], ["Carol", 92]]

with open("scores.csv", "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerow(["Name", "Score"])   # header
    writer.writerows(rows)               # all data rows at once

Relative paths in a repo

Hard-coding /Users/yourname/... breaks the moment you share your code, run it on a server, or submit to GitHub. Always use paths relative to your project root. The convention in this course is to put data files in a data/ subfolder within the module folder.

If you need to find the directory your script lives in (for truly portable path construction), use os.path.dirname(__file__) from the os module. But for this course, a simple relative string like "data/sales.csv" is sufficient as long as you run your script from the project root.

# BAD — only works on your machine
open("/Users/alice/projects/ism2411/data.csv")

# GOOD — works for anyone who clones the repo
open("data/sales.csv")

# BEST for production — dynamically finds the script's directory
import os
script_dir = os.path.dirname(__file__)
data_path = os.path.join(script_dir, "data", "sales.csv")
open(data_path)

Code Walkthrough

csv.DictReader — the more readable approach

csv.DictReader reads each row as a dictionary using the first row as column names. This is much cleaner for files with many columns — you access fields by name (row["price"]) instead of by index (row[2]). It also automatically skips the header row, so you don't need next(reader).

import csv

totals = []

with open("data/sales.csv", "r") as f:
    reader = csv.DictReader(f)
    for row in reader:
        # Access by column name — much more readable
        product = row["product"]
        price   = float(row["price"])
        qty     = int(row["quantity"])
        total   = price * qty

        totals.append(total)
        print(f"{product}: {qty} units × ${price:.2f} = ${total:.2f}")

print(f"\nTotal revenue: ${sum(totals):,.2f}")
print(f"Average order: ${sum(totals)/len(totals):,.2f}")

Complete ETL pipeline: read, transform, write

ETL — Extract, Transform, Load — is the standard term for "read data, do something to it, write the result." Every data pipeline in business follows this pattern. Here is a complete example that reads a sales file, computes totals, filters for high-value orders, and writes a cleaned output.

import csv

# --- EXTRACT ---
with open("data/sales.csv", "r") as f:
    reader = csv.DictReader(f)
    orders = list(reader)   # load all rows into memory

# --- TRANSFORM ---
for order in orders:
    order["total"] = float(order["price"]) * int(order["quantity"])

high_value = [o for o in orders if o["total"] > 100]

# --- LOAD ---
with open("data/cleaned.csv", "w", newline="") as f:
    fieldnames = ["product", "price", "quantity", "total"]
    writer = csv.DictWriter(f, fieldnames=fieldnames)
    writer.writeheader()
    writer.writerows(high_value)

print(f"Wrote {len(high_value)} / {len(orders)} orders to cleaned.csv")

Common Errors

TypeError when computing from CSV values

Every value from csv.reader is a string. Multiplying strings does not do arithmetic — it repeats them. Convert before computing.

# WRONG — "9.99" * "50" raises TypeError
revenue = row[1] * row[2]

# RIGHT — convert first
revenue = float(row[1]) * int(row[2])

FileNotFoundError from wrong path

Python looks for the file relative to the current working directory — where you ran the script from, not where the script lives. If you run python week12/solution.py from the project root, then "data/sales.csv" means project_root/data/sales.csv. If you cd week12 first and then run python solution.py, then "data/sales.csv" means week12/data/sales.csv. Be consistent about your working directory.

# Debug: print your current directory so you know where Python is looking
import os
print(os.getcwd())

Extra blank rows in Excel from Windows line endings

When writing CSV files, always use newline="" in the open() call. Without it, Python adds \r\n on Windows, and the csv module also adds its own, resulting in double line endings that appear as blank rows.

# WRONG on Windows — blank rows in Excel
with open("output.csv", "w") as f:
    writer = csv.writer(f)

# RIGHT — correct on all platforms
with open("output.csv", "w", newline="") as f:
    writer = csv.writer(f)

Key Takeaways
  • Always use with open(...) as f: — it closes the file automatically and is safer than manual close.
  • "w" mode clears the file immediately; use "a" to append, not "w".
  • CSV values are always strings — convert with float() and int() before arithmetic.
  • Use csv.DictReader for readable column access by name; use csv.reader for simple sequential access.
  • Use relative paths — "data/sales.csv" not "/Users/yourname/..." — so your code runs on any machine.
