What is a Computer?

Understanding what your computer actually does when you double-click an Excel file.

Module 01 Unit 1 Reading · Think Python (TP) Ch. 1 — Programming as a Way of Thinking ★ Midterm-Eligible
This module: Reading Lecture Lab

Why this matters

Most people who write code never learned what a computer actually does. They learn syntax, and then bugs feel like magic. This module we get the mental model in place — CPU, memory, storage, files, paths — so that every Python concept we meet afterward has something concrete to attach to. Without this foundation, every error message is a mystery. With it, you can read an error, reason about what the machine was trying to do, and fix it systematically.

Think about the last time you opened an Excel spreadsheet. The file was sitting on your hard drive, dormant. The moment you double-clicked it, Windows or macOS executed a cascade of events: the operating system read bytes off storage, copied them into RAM, launched the Excel process, and handed control to the CPU. All of that happened in under a second — silently, invisibly. Every Python script you write this semester triggers exactly the same cascade.

Business analysts who understand hardware think differently. They know why a script that processes 50,000 rows runs fast but one that processes 5 million rows needs optimization. They know why saving a file is different from having a file open. They understand that "the program crashed" means something went wrong in RAM, while "the file is corrupt" means something went wrong in storage. These distinctions matter the moment your code hits production data.

Core Concepts

These five terms appear in every conversation about computers. Nail them now and they will serve you for your entire career.

CPU

The Central Processing Unit is the brain of the computer. It executes instructions one at a time, billions of times per second. Think of it as a chef who can only hold one recipe card at a time but flips through them impossibly fast.

RAM

Random Access Memory holds whatever the computer is currently working on. It is fast but temporary — when you lose power, RAM is wiped. Opening a file copies it from storage into RAM so the CPU can touch it.

Storage

Your hard drive or SSD holds files permanently, even when power is off. It is slower than RAM but survives reboots. Think of RAM as your desk and storage as your filing cabinet.

File

A file is a named, ordered sequence of bytes stored on disk. The bytes can represent text, images, spreadsheet data, or compiled code. The file extension (.xlsx, .py, .csv) hints at the encoding.

Path

A path is the complete address of a file in the file system, like a street address for data. /home/alice/reports/q1.csv unambiguously locates one file across an entire drive with billions of files.

Business Scenario

Imagine you are a financial analyst at a regional retailer. Every Monday morning, someone emails you a CSV export from the point-of-sale system. You open it in Excel, do some cleanup, paste numbers into a PowerPoint slide, and send it to your manager. The whole process takes about 90 minutes. Your colleague in the IT department runs a Python script instead — it reads the CSV, computes the same numbers, generates the slide, and emails it automatically. The script runs in under 10 seconds.

The difference is not intelligence — it is knowing how the machine works. The Python script navigates to the file using its path, reads bytes from storage into RAM, manipulates them with CPU instructions, and writes a new file back to storage. Every step in that pipeline maps directly to concepts in this module. Once you can see the pipeline, you can build it.

By the end of this course you will be able to write that script. This module is the first building block: understanding what is actually happening under the hood when any program runs.

Key Vocabulary

  • CPU (Central Processing Unit) — the chip that executes instructions; the "brain" of the computer
  • RAM (Random Access Memory) — fast, temporary memory where active programs and open files live
  • Storage — persistent memory (HDD or SSD) where files survive power-off
  • File — a named, ordered collection of bytes saved to storage
  • Directory (Folder) — a container that organizes files and other directories in a hierarchy
  • File path — the full address of a file, from root to filename (e.g., /Users/alice/data/sales.csv)
  • Extension — the suffix after the last dot in a filename that signals the file's format (.py, .csv, .xlsx)
  • Interpreter — a program that reads source code (like Python) line-by-line and tells the CPU what to do
  • Operating System (OS) — software (Windows, macOS, Linux) that manages hardware resources and runs your programs
Common Misconceptions
  • "RAM and storage are the same thing." They are not. RAM is fast and temporary; storage is slow and permanent. A 16 GB laptop can open a 2 GB file because it copies it into RAM — but if you have 10 files open simultaneously, you may run out of RAM.
  • "Saving a file writes it to RAM." The opposite: pressing Ctrl+S moves data FROM RAM TO storage. Until you save, your edits exist only in RAM and will be lost if power is cut.
  • "A .py file is a special kind of file." It is plain text. You can open any .py file in Notepad and read it. The extension is a convention that tells the Python interpreter what to do with it.

Module Quiz ★ Midterm-Eligible

Q1 · Multiple Choice
Which component of a computer executes program instructions?
  • a) Hard drive
  • b) RAM
  • c) CPU
  • d) Monitor
Answer
CPU — CPU = Central Processing Unit
Q2 · Multiple Choice
What is a file path?
  • a) Transfer speed
  • b) The location of a file in the file system
  • c) File size
  • d) File extension
Answer
The location of a file in the file system
Q3 · Multiple Choice
Which file extension is used for Python programs?
  • a) .txt
  • b) .doc
  • c) .py
  • d) .xls
Answer
.py
Q4 · Multiple Choice
What does a directory contain?
  • a) Only files
  • b) Only other directories
  • c) Files and/or other directories
  • d) CPU instructions
Answer
Files and/or other directories
Q5 · Multiple Choice
Which best describes what a program is?
  • a) A physical chip
  • b) A sequence of instructions that tells a computer what to do
  • c) A storage device
  • d) An operating system
Answer
A sequence of instructions that tells a computer what to do
Q6 · Multiple Choice
When you open an Excel file, where is the file's data loaded so the CPU can work with it?
  • a) The hard drive
  • b) The CPU cache
  • c) RAM
  • d) The monitor
Answer
RAM — the OS copies data from storage into RAM so the CPU can access it quickly
Q7 · True / False
A .py file is a special binary format that only the Python interpreter can read.
Answer
False — a .py file is plain text. Any text editor can open it. The .py extension is a convention that tells the interpreter how to process it.
Q8 · Short Answer
What is the key difference between RAM and storage (hard drive / SSD)?
Answer
RAM is fast and temporary (contents lost when power is off). Storage is slower but persistent (contents survive power-off). Files live on storage; active programs run in RAM.
