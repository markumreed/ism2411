Why this matters

The terminal is the place where everyone who works in data and engineering meets. GUIs change every five years; the shell does not. The five commands you learn this module are the same five you will use in 20 years. Every cloud server, every remote machine, every automated pipeline runs through a command line interface. The sooner you make it feel like home, the faster everything else clicks.

There is a deeper reason too. When you click buttons in a GUI application, the application is issuing shell commands on your behalf — you just cannot see them. The terminal removes the middleman. You issue commands directly to the OS. This means you can do things no GUI can: automate sequences, pipe output from one command into another, schedule scripts to run at midnight, and process files that are too large to open in any desktop app.

Business analysts who are comfortable at the command line have a compounding advantage. They can navigate to a project folder in three keystrokes instead of clicking through eight nested folders. They can verify that a script ran and produced the right output file without opening a file manager. They can install new Python packages in seconds and check which version of Python is installed without digging through settings menus. This module gives you that power.

Core Concepts

Terminal / Shell

A text interface where you type commands and the OS executes them. macOS uses zsh, Linux uses bash, Windows uses PowerShell or cmd. All share the same core navigation commands covered this module.

Working Directory

The folder the shell is currently "inside." Every command you run operates relative to this location. pwd tells you where you are; cd moves you. Think of it as your current position on a map.

Absolute Path

A path that starts from the root of the file system: /home/alice/projects/sales.csv. It works from anywhere because it is fully specified — like a complete street address including city and country.

Relative Path

A path relative to your current working directory: data/sales.csv. Shorter and portable across machines, but only valid if you are in the right directory. ./ means "starting here"; ../ means "go up one level."

pip

Python's package manager. pip install pandas downloads and installs the pandas library from the internet. Think of it as an app store for Python modules — one command to add new capabilities.

Business Scenario

A sales operations analyst at a mid-size distributor needs to produce a weekly summary from a CSV file that the ERP system drops into /data/exports/ every Friday at 6 PM. Previously she would open File Explorer, navigate to the folder, double-click the CSV, do calculations in Excel, and copy the results into an email — a 45-minute process. After this module, she writes a Python script called weekly_summary.py in VS Code, saves it in her project folder, and runs it from the terminal with python3 weekly_summary.py.

The script finds the file using its path, reads the data, computes revenue and units sold by region, and prints a formatted summary. Running the script takes eight seconds. She schedules it with a cron job so the summary appears in her inbox automatically every Friday at 6:10 PM. The 45-minute manual process is gone. That automation begins with exactly the skills in this module: knowing where your files are, how to navigate there, and how to run a script.

Key Vocabulary

  • Terminal (Shell) — a text-based interface for issuing commands to the operating system
  • pwd — "print working directory"; shows the full path of your current location
  • ls — lists files and folders in the current directory (use dir on Windows cmd)
  • cd — "change directory"; moves you into a different folder (cd .. goes up one level)
  • mkdir — "make directory"; creates a new folder
  • Absolute path — full path from the root: /home/user/data/file.csv
  • Relative path — path from your current directory: data/file.csv or ../other/file.csv
  • Python interpreter — the program that reads your .py file and executes it line by line
  • pip — Python's package installer; use pip install packagename to add libraries
Common Misconceptions
  • "cd takes me to the file, not the folder." You can only cd into directories, not files. Navigate to the folder containing the file, then reference the file by name.
  • "Relative paths are always shorter." Not always — if you are deep in one branch and the file is in another branch, the relative path with ../../ can be longer than an absolute path. Use whichever is clearer.
  • "python and python3 are the same." On many systems, python runs Python 2 (old, deprecated) and python3 runs Python 3 (current). Always use python3 or confirm your setup with python --version.

Module Quiz ★ Midterm-Eligible

Quiz questions repeat on the exam verbatim or with minor variation. Click each answer to reveal it.

Q1 · Multiple Choice
Which command prints the current working directory?
  • a) cd
  • b) ls
  • c) pwd
  • d) mkdir
Answer
pwd — pwd = print working directory
Q2 · Multiple Choice
What does the cd command do?
  • a) Creates a directory
  • b) Lists files
  • c) Changes the current directory
  • d) Deletes a file
Answer
Changes the current directory
Q3 · Multiple Choice
Which of the following is an absolute path?
  • a) ../data/sales.csv
  • b) data/sales.csv
  • c) /home/user/data/sales.csv
  • d) ./sales.csv
Answer
/home/user/data/sales.csv — Absolute paths start from the root
Q4 · Code Trace
What does this terminal sequence do?
mkdir project
cd project
pwd
Answer
Creates a directory called 'project', moves into it, then prints its full path
Q5 · Multiple Choice
What file extension do Python scripts use?
  • a) .pt
  • b) .py
  • c) .python
  • d) .pyt
Answer
.py
Q6 · Multiple Choice
What does cd .. do in the terminal?
  • a) Creates a new directory
  • b) Moves up one level to the parent directory
  • c) Lists hidden files
  • d) Clears the screen
Answer
Moves up one level to the parent directory — .. always refers to the parent of the current directory
Q7 · Short Answer
What is the difference between a relative path and an absolute path? Give one example of each.
Answer
An absolute path starts from the file system root and works from anywhere: /home/alice/data/sales.csv. A relative path starts from the current working directory: data/sales.csv. The relative path only works if you are in the right directory.
Q8 · Code Trace
You are in /home/alice/projects/. What does cd ../data do?
Answer
It moves up to /home/alice/ (the .. step) and then into the data directory there, landing at /home/alice/data/.
