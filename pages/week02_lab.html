Run your first commands in the terminal, install Python and VS Code, and execute hello.py.

Exercises

EXERCISE 1 BASIC
Terminal warm-up
Open your terminal. Run: pwd, ls, then mkdir ism2411 and cd into it. Run pwd again — confirm you are inside the new folder.
Expected: second pwd output shows the full path ending in /ism2411
EXERCISE 2 BASIC
Verify Python
Run python3 --version. You should see a version 3.10 or higher. If not, see the precourse setup page. Also run pip3 --version to confirm pip is installed.
Expected: output like Python 3.11.4 and pip 23.x.x
EXERCISE 3 BASIC
Write hello.py
In VS Code, create a file called hello.py with the line: print("Hello, ISM2411!"). Save it. From the terminal, in the same folder, run: python3 hello.py
Expected: Hello, ISM2411! printed in the terminal
EXERCISE 4 BASIC
Path practice
From your ism2411 folder, navigate one level up (cd ..), then back in (cd ism2411). Practice cd-ing between folders without using the GUI. Run pwd after each cd to confirm your location.
Expected: pwd alternates between parent and ism2411 paths as you navigate
EXERCISE 5 BASIC
Make a sub-project
Inside ism2411, run: mkdir module02, cd module02, then create a second hello.py that prints your name and major. Run it.
Expected: output like Alex Chen — Finance major
EXERCISE 6 INTERMEDIATE
Deliberate error experiment
Introduce a typo in your hello.py — for example, change print to primt or remove a closing parenthesis. Run the script and read the error message carefully. Write down: (a) the line number reported, (b) the type of error, (c) what the message tells you about the problem. Then fix it.
Expected: a NameError or SyntaxError with a line number; written analysis of the error message structure
EXERCISE 7 INTERMEDIATE
Run from the wrong directory
Navigate one level up from module02. Try to run your hello.py with: python3 hello.py. What error do you get? Now run it with the correct relative path: python3 module02/hello.py. Note the difference in behavior and explain why.
Expected: first attempt fails with "No such file or directory"; second attempt succeeds

Stretch Challenge

STRETCH A CHALLENGE
Multi-print script
Write a script called intro.py that prints at least five lines including: your name, your major, your hometown, one hobby, and why you are taking this course. Format each line with a label, like: Name: Alex Chen. Run it from the terminal.
STRETCH B CHALLENGE
Install and test a package
Run pip3 install requests in your terminal. Then create a file called test_requests.py with the line import requests; print("requests version:", requests.__version__). Run it. If it prints the version number, requests is installed correctly — you will use this package later in the semester.

Reflection

Self-Assessment Questions
  1. Before today, how did you think about navigating your computer? Has using only the terminal (no GUI) changed your mental model of where files live?
  2. When you encountered your first error in Exercise 6, what was your instinct? Did you read the error message carefully, or were you tempted to just start changing things randomly? What does this tell you about debugging habits?
  3. What would you need to do differently to run a Python script reliably on a collaborator's machine — someone with a different username and a different folder structure?
