Conditionals

Conditionals let your code make decisions. This unlocks the difference between a calculator and an actual program — code that does something different depending on the data.

Module 05 · Lecture Unit 1 75-min session
This module: Reading Lecture Lab

Today's walkthrough

The basic if / elif / else

Python uses indentation (4 spaces) to show which lines belong to the if. The colon at the end of the if line is required — omitting it is a SyntaxError. Unlike some languages, Python does not use braces or parentheses to delimit blocks; the indentation is the block.

The if/elif/else chain evaluates conditions in order. As soon as one condition is True, Python executes that block and jumps past all the remaining branches. This means if you order your conditions incorrectly, you can get logic bugs that are hard to spot — for example, putting a broader condition before a narrower one.

The else clause is optional and acts as a catch-all. If none of the if or elif conditions are true, the else block runs. If there is no else and no conditions match, Python simply moves on — no error, just silence.

score = 85
if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "F"
print(grade)  # B

A discount-tier example

A classic business pattern: the discount depends on the total. Notice that the conditions are ordered from highest to lowest. If you wrote if total >= 100: first, every order over $100 would get a 5% discount and never reach the higher tiers — a costly logic bug.

This is a pattern you will see in pricing engines, payroll calculators (tax brackets work exactly this way), and loyalty tier assignments. The structure is always the same: establish ranges from largest to smallest, return the first match.

After applying the discount, we use an f-string to format the output with two decimal places — a small detail that matters a lot when you are displaying monetary values to customers or finance teams.

total = 250

if total >= 500:
    discount = 0.15
elif total >= 200:
    discount = 0.10
elif total >= 100:
    discount = 0.05
else:
    discount = 0

final = total * (1 - discount)
print(f"Total ${total} → discount {discount*100:.0f}% → pay ${final:.2f}")

Nested conditionals (and when to avoid them)

You can put an if inside another if. It works — but readability suffers fast. Beyond two levels of nesting, most experienced developers will find the code hard to follow. Deeply nested conditionals are also harder to test, because each level doubles the number of paths through the code.

The preferred approach is to flatten nested conditions using and or or. This keeps all the decision logic on a single level and makes it immediately clear what combination of conditions leads to each outcome.

There are cases where nesting is genuinely necessary — for example, when the inner condition is only meaningful after the outer condition has been confirmed. But as a rule, if you find yourself writing three or more levels deep, pause and ask whether you can restructure.

# nested — gets ugly fast
if region == "South":
    if total > 100:
        discount = 0.1

# flat — much easier to read
if region == "South" and total > 100:
    discount = 0.1

Common mistake: = vs. ==

A single = is assignment ("set this variable to this value"). A double == is comparison ("are these two values equal?"). In Python, using = inside an if condition is a SyntaxError, which is actually helpful — in some older languages it was a silent bug. Still, confusing them is the #1 beginner mistake, so it is worth drilling until it is automatic.

Related: use != for "not equal to", not <> (which is invalid in Python 3). And remember that >= means "greater than or equal to" — it includes the boundary value, which matters for discount tiers where $500 exactly should qualify for the top tier.

# WRONG — this assigns, does not compare
if x = 5:
    print("five")

# RIGHT
if x == 5:
    print("five")

Detailed Code Walkthrough

Walkthrough 1: Approval workflow

This example shows a two-stage decision: first determine the discount, then check whether approval is required. Each stage is an independent if/elif/else block. Notice how readable it is compared to a nested version would be.

# Stage 1: determine discount
order_total = 750

if order_total >= 500:      # Line 3: is total at or above $500?
    discount_pct = 0.15     # Line 4: yes — 15% discount
elif order_total >= 200:    # Line 5: otherwise, at or above $200?
    discount_pct = 0.10     # Line 6: yes — 10% discount
else:                       # Line 7: none of the above
    discount_pct = 0        # Line 8: no discount

# Stage 2: compute final and check approval threshold
discounted = order_total * (1 - discount_pct)   # $637.50

if discounted > 1000:                           # approval threshold
    approval = "manager_required"
else:
    approval = "auto_approved"

print(f"Discounted total: ${discounted:.2f}")
print(f"Approval status: {approval}")

Walkthrough 2: Truthy and falsy in practice

This pattern is very common — checking whether a list or string is empty before processing it. Using truthiness keeps the code concise and Pythonic.

cart_items = []          # empty list — falsy

if cart_items:           # evaluates to False because list is empty
    print("Processing order...")
else:
    print("Cart is empty — nothing to do.")

# Compare: verbose version that does the same thing
if len(cart_items) != 0:
    print("Processing order...")
else:
    print("Cart is empty — nothing to do.")

# The first version is preferred Python style

Common Errors

ERROR 1 — Missing colon
SyntaxError: expected ':'
# WRONG
if score >= 90
    print("A")

# RIGHT — colon is required after every condition
if score >= 90:
    print("A")
ERROR 2 — Wrong condition order
Logic bug: broader condition catches everything first
# WRONG — every order >= 100 gets 5%, never reaches higher tiers
if total >= 100:
    discount = 0.05
elif total >= 200:
    discount = 0.10
elif total >= 500:
    discount = 0.15

# RIGHT — order from most specific (highest) to least specific
if total >= 500:
    discount = 0.15
elif total >= 200:
    discount = 0.10
elif total >= 100:
    discount = 0.05
ERROR 3 — Indentation error
IndentationError: expected an indented block
# WRONG — body of if must be indented
if score >= 90:
print("A")

# RIGHT — 4 spaces of indentation
if score >= 90:
    print("A")

Discussion Questions

  1. In what business scenario might the order of conditions in an elif chain lead to a financially significant bug? Describe a realistic example and explain how you would test for it.
  2. Python allows conditions like if 0 < total < 500: (chained comparisons). How does this compare in readability to if total > 0 and total < 500:? When would you prefer each style?
  3. Suppose you are writing a loan approval system with five criteria (credit score, income, debt ratio, loan amount, employment status). Should you use deeply nested ifs, or a flat approach with and? What are the trade-offs?
  4. How do conditionals in Python differ from Excel's IF function? What can Python do that Excel cannot, and vice versa?

Key Takeaways
  • Python evaluates if/elif/else conditions top to bottom and executes only the first matching branch.
  • Order your conditions from most specific (highest threshold) to least specific (lowest threshold).
  • Use == for comparison, = for assignment — never mix them up in a condition.
  • Truthy/falsy rules let you write concise checks like if items: instead of if len(items) > 0:.
  • Flatten nested ifs with and/or whenever possible — your future self will thank you.
