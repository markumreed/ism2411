Operators & Expressions

Calculating revenue, profit margin, and discounted price using Python's operator toolkit.

Why this matters

Operators are the verbs of Python. You already know the math ones; the new ideas are comparison (returns True/False) and logical (combines comparisons). Business formulas are nothing more than operators applied to variables. Revenue is price multiplied by quantity. Margin is the difference between price and cost divided by price. Every KPI dashboard, every financial model, and every sales report you will ever build ultimately reduces to a set of operator expressions applied to stored data.

The less obvious operators — // (floor division), % (modulo), and ** (exponentiation) — show up more often than beginners expect. Floor division is useful when you need a whole-number result: how many full boxes of 24 fit in an order of 100 units? Modulo tells you the remainder: what is left over after filling those boxes? These two operators together can handle any packing, batching, or scheduling problem that involves discrete units.

Order of operations matters as much in Python as it did in algebra. The rule is PEMDAS: Parentheses, Exponents, Multiplication/Division, Addition/Subtraction. Forgetting this causes silent bugs — your formula runs without error but produces the wrong number. Adding explicit parentheses even when not strictly required is a professional habit that makes code easier to read and audit.

Core Concepts

Arithmetic Operators

+ - * / work as expected. // is floor division (result is an int). % is modulo (the remainder). ** is exponentiation: 2**10 is 1024.

Comparison Operators

== (equal), != (not equal), >, <, >=, <=. Every comparison returns a boolean (True or False). Comparison is how programs make decisions.

Logical Operators

and requires both sides to be True. or requires at least one side to be True. not flips the boolean. Use them to combine conditions: price > 100 and in_stock.

Order of Operations

Python follows PEMDAS. 2 + 3 * 4 is 14, not 20, because * runs before +. Use parentheses to be explicit: (2 + 3) * 4 is 20. When in doubt, add parentheses.

Type Mismatch

"5" + 5 raises a TypeError because you cannot add a string to an integer. Convert first: int("5") + 5 gives 10. This is a classic source of errors when processing user input.

Business Scenario

A pricing analyst at an e-commerce company needs to evaluate a promotional strategy. The company is considering a 15% discount on products with margin above 40%, and a 7% sales tax will be applied after the discount. She has three products with different prices and costs and needs to compute net revenue after discount and tax for each.

For a product priced at $80 with a cost of $45: margin = (80 - 45) / 80 gives 0.4375 (43.75%). Since margin > 0.40 is True, the discount applies: discounted_price = 80 * (1 - 0.15) gives $68. After 7% tax: final_price = 68 * (1 + 0.07) gives $72.76. The script computes all of this in a few lines — and can be updated instantly when the discount rate changes. That is the power of encoding business logic as operators and expressions.

Notice how the parentheses in (1 - 0.15) and (1 + 0.07) are crucial. Without them, order of operations would yield a different, incorrect result. This is exactly where business Python bugs are born: silent arithmetic errors caused by wrong precedence assumptions.

Key Vocabulary

  • Operator — a symbol that performs an operation on one or more values (+, -, *, /, ==, etc.)
  • Operand — a value on which an operator acts; in 3 * 4, the operands are 3 and 4
  • Expression — a combination of values and operators that evaluates to a single value
  • Floor division (//) — divides and rounds down to the nearest integer: 17 // 5 is 3
  • Modulo (%) — returns the remainder after division: 17 % 5 is 2
  • Exponentiation (**) — raises to a power: 2 ** 8 is 256
  • Comparison operator — returns True or False: == != > < >= <=
  • Logical operator — combines boolean values: and, or, not
  • TypeError — raised when an operation is applied to incompatible types (e.g., "5" + 5)
  • PEMDAS — mnemonic for Python's order of operations: Parentheses, Exponents, Multiplication/Division, Addition/Subtraction
Common Misconceptions
  • "/ and // do the same thing." They do not. 10 / 3 returns 3.3333... (a float). 10 // 3 returns 3 (an integer, the floor). Use // only when you specifically need a whole number result.
  • "You can add a string and a number." Python raises a TypeError: "5" + 5 fails. You must convert explicitly: int("5") + 5. This trips up beginners who combine input() values with numeric variables.
  • "and and or return True or False." Actually they return one of their operands. 5 and 10 returns 10; 0 or 7 returns 7. In boolean contexts this is fine, but it can surprise you when you print the result.

What error does this raise, and why?

print(total)
Answer
If the user types a number like "3", the result is NOT 15 — it is "33333" (string repetition). input() returns a str, so "3" * 5 repeats the string 5 times. Fix: int(input("Enter quantity: ")) * 5.
