Exercises

EXERCISE 1
BASIC
Discount tiers
Write a function-free script: ask the user for a cart total. Apply: 15% off if >= $500, 10% off if >= $200, 5% off if >= $100, otherwise no discount. Print the final price.

STARTER CODE

total = float(input("Enter cart total: $"))
# Add your if/elif/else here
discount = 0  # replace with correct logic
final = total * (1 - discount)
print(f"Discount: {discount*100:.0f}%")

EXPECTED OUTPUT (for total = 250)

Discount: 10%
Final price: $225.00
EXERCISE 2
BASIC
Approval flag
Add a second decision: if final price > $1000, set status = "manager_approval_required", else status = "auto_approved". Print the status.

EXPECTED OUTPUT (for final = $637.50)

Status: auto_approved
EXERCISE 3
INTERMEDIATE
Combine conditions
Modify so that South-region orders get an extra 2% off on top of their tier discount. Use and to combine the region check with the total check. Test with region = "South" and total = 250.

EXPECTED OUTPUT

Region bonus applied: additional 2% off
Final discount: 12%
Final price: $220.00

EXERCISE 5
INTERMEDIATE
Anomaly flagging
Given a transaction amount, print "NORMAL" if between $1 and $9,999, "SUSPICIOUS" if $0 or negative, and "LARGE — REVIEW REQUIRED" if $10,000 or more. Test with values -50, 500, and 15000.

EXPECTED OUTPUT (for amount = 15000)

LARGE — REVIEW REQUIRED
EXERCISE 6
INTERMEDIATE
Boolean logic
A customer qualifies for a "VIP upgrade" if they have spent more than $2000 total this year OR have placed more than 10 orders. Write the condition. Then add a second rule: they are excluded if their account has any outstanding balance. Use not.

Stretch Challenge

STRETCH 1 — CHALLENGE
Full pricing engine
Build a script that takes: cart total, customer region, customer type (retail/wholesale), and whether this is their first order. Apply rules in this order: (1) tier discount, (2) regional bonus if South, (3) wholesale extra 5%, (4) first-order 3% welcome discount. Compute and display the final price and a line-by-line breakdown of each discount applied.
STRETCH 2 — CHALLENGE
US federal income tax brackets (simplified)
For a single filer in 2024: 10% on income up to $11,600; 12% on $11,601–$47,150; 22% on $47,151–$100,525. Write an if/elif script that computes the marginal tax for a given income. Note: this is marginal taxation, not flat — each bracket only applies to the income within that range.

Reflection

Self-Assessment

1. Without looking at your code, can you explain in plain English what happens to order_total = 350 as it passes through your discount tier logic? Trace it step by step.

2. What is the difference between if total > 100 and total < 500: and two separate if statements checking each condition? Would both produce the same result? Why or why not?

3. Where in your day-to-day life do you encounter systems that use tiered conditional logic? (Think: tax brackets, shipping fees, insurance premiums, credit card rewards.) How does knowing Python conditionals change how you think about those systems?
