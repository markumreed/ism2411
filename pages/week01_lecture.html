Today's walkthrough

What is a computer, really?

A computer is three things working together: a CPU that follows instructions one at a time, RAM that holds what it is currently working on, and storage where files live when the power is off. When you open a file, the OS copies it from storage into RAM and the CPU can act on it. Understanding these three layers is the foundation of everything you will do in this course — every Python script you write will execute in this same pipeline.

Think of the CPU as the chef, RAM as the kitchen counter, and storage as the pantry. The chef can only work with what is on the counter. To use an ingredient from the pantry (storage), you first carry it to the counter (RAM), then the chef (CPU) works with it. When the chef is done and the kitchen closes (power off), everything on the counter disappears — but the pantry is still stocked. This analogy explains why "save your work" is not just good advice; it is the difference between data in volatile RAM and data on permanent storage.

Modern computers complicate this picture with CPU caches, virtual memory, and SSDs that blur the line between RAM and storage. But for programming purposes, the three-layer model is accurate enough to reason about almost every bug and performance issue you will ever encounter. When a program crashes, something went wrong in RAM. When a file is corrupt, something went wrong in storage. When a program is slow, you are probably waiting on storage I/O.

Files, paths, directories

A file is a named blob of bytes. A directory (also called a folder) is a labeled container that holds files and other directories. Every file has a path — the address that tells the system how to get to it. Like a street address, it works because it is unambiguous. The path /home/alice/reports/q1.csv describes exactly one location in the entire file system, no matter where you are standing.

Paths work hierarchically. On macOS and Linux, the hierarchy starts at / (the root). On Windows, it starts at a drive letter like C:\. Everything else — every folder and file on the system — is nested inside that root. A relative path describes a location starting from wherever you currently are, not from the root. This distinction matters when you write Python scripts that open files: use relative paths so your code works on any machine, not just yours.

C:\Users\you\Documents\report.xlsx       # Windows absolute
/home/you/Documents/report.xlsx          # Mac/Linux absolute
./data/sales.csv                         # relative (from current folder)
../reports/summary.xlsx                  # relative (up one level, then in)

What is a program?

A program is a sequence of instructions written in a language the computer can run. Python is one such language. When you run a Python program, an interpreter reads each line and tells the CPU what to do. The interpreter is itself a program — written in C — that translates Python instructions into the machine code the CPU understands. This is why Python is called an "interpreted language."

The alternative is a "compiled language" like C++ or Java, where a compiler translates the entire source file into machine code before any of it runs. Compiled programs start faster but take longer to build. Interpreted programs start immediately (the interpreter is already running) but execute more slowly. For business data tasks — analyzing thousands of rows, generating reports, calling APIs — Python's execution speed is more than adequate, and its development speed is far superior.

A .py file is just a text file. Open any Python script in Notepad and it is perfectly readable. The .py extension is a convention that signals to the OS and your editor: "this is Python code." When you run python3 script.py, you are asking the Python interpreter program to read and execute that text file, line by line.

Why businesses use programming

Excel works fine for hundreds of rows. It breaks down at millions of rows, across dozens of files, when work needs to be repeated weekly, or when it needs to be checked by someone else. Code is reusable, auditable, and version-controlled. That is the value. A Python script that runs every Monday morning and emails a summary requires no human interaction after it is written. An Excel process requires someone to open a file, run a macro, copy a number, paste it somewhere else, and hope they did not accidentally edit a formula.

Version control is another advantage that non-programmers underestimate. With Git (which you will encounter later this semester), every change to a script is tracked. If a bug is introduced, you can see exactly what changed and when. An Excel file with "v3_FINAL_revised_USE_THIS_ONE.xlsx" in the name is version "control" by comparison. Programming forces you to be explicit about what the code does — and that explicitness is auditable, which matters enormously in regulated industries like finance and healthcare.

Common Errors

Confusing RAM and storage

Students often ask: "I saved my file, so why did my changes disappear?" Saving writes from RAM to storage. If you closed a file without saving — or the program crashed before writing — the RAM contents were lost. The solution is always: save early, save often, and understand that "open" means "in RAM."

Confusing absolute and relative paths

A script that opens /home/alice/data/sales.csv (absolute) only works on Alice's machine. The same script using data/sales.csv (relative) works on anyone's machine — as long as they are in the right directory when they run it. This is a day-one mistake that wastes hours later. Get in the habit of using relative paths in code.

Discussion Questions

  1. Think about a repetitive task in your current job or internship that involves copying and pasting between files. Describe the pipeline of CPU, RAM, and storage operations happening while you do that task.
  2. What is the difference between an interpreted language and a compiled language? Why might a business analyst prefer Python over a compiled language for data work?
  3. If a hard drive fails, you lose your files. If RAM fails, you lose your currently open work. Which type of failure is more catastrophic for a business, and what practices mitigate each?
  4. Why is version control important in a business setting? Give an example of a problem version control would prevent that is common with shared Excel files.

Key Takeaways
  • A computer is CPU + RAM + Storage working together. Each layer has a distinct role.
  • RAM is fast and temporary; storage is slower and permanent. Save = write from RAM to storage.
  • A file is bytes + a name. A path is the full address of that file in the directory hierarchy.
  • A program is a sequence of instructions; an interpreter executes Python programs line by line.
  • Businesses use programming because code is reusable, auditable, and scales beyond what manual processes can handle.
