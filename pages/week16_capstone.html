Capstone — Retail Sales Analysis

A single sustained project using a real-shaped retail sales dataset of about 5,000 rows. You will clean it, ask three to five business questions, answer them with code and visualizations, and present your findings in class. This is the deliverable you will add to your portfolio.

Module 16 Unit 5 Capstone — 25% of grade
What this is: A practical demonstration that you can take a CSV file no one has explained to you, figure out what it contains, decide what is interesting, and tell that story clearly with code and charts. This is the exact skill employers are paying for when they hire an analyst straight out of a business program.

The dataset

You receive a file called retail_sales.csv at the start of Module 13. It contains roughly 5,000 rows of fictional retail transactions with these columns:

  • order_date — date of the sale (mostly clean, with some bad values that need errors="coerce")
  • product_category — one of about 8 categories (with some typos: "Electronix", "furnitu re")
  • region — one of North, South, East, West (some nulls)
  • unit_price — dollar amount per item stored as a string with "$" prefix (some negatives = returns)
  • quantity_sold — units in the order (some zeros, some missing)
  • customer_segment — Consumer, Corporate, or Home Office

The data is intentionally messy. Cleaning it well — and documenting your decisions — is half the assignment. A messy dataset that produces wrong statistics is worse than no dataset at all.

df = pd.read_csv("data/retail_sales.csv")
print("Raw shape:", df.shape)
print(df.head())
print(df.info())
print(df.isnull().sum())

# ── 2. CLEAN ──────────────────────────────────────────────────
# Document every decision with a comment explaining WHY.

# Standardize column names
df.columns = df.columns.str.lower().str.replace(" ", "_")

# Drop rows missing essential identifiers
# Reason: cannot attribute revenue without an order date
df = df.dropna(subset=["order_date"])

# Parse order_date — coerce bad dates to NaT
df["order_date"] = pd.to_datetime(df["order_date"], errors="coerce")

# Fix unit_price: strip "$", convert to float
df["unit_price"] = df["unit_price"].str.replace("$","",regex=False).astype(float)

# Exclude returns (negative unit_price)
# Reason: question focuses on forward sales; returns analyzed separately
df = df[df["unit_price"] >= 0]

# Fill missing quantity_sold with median (low %, aggregate impact minimal)
df["quantity_sold"] = df["quantity_sold"].fillna(df["quantity_sold"].median())

# Fix category typos
df["product_category"] = df["product_category"].str.strip().str.title()

# Remove duplicates
df = df.drop_duplicates()
print("Clean shape:", df.shape)

# ── 3. DERIVED COLUMNS ────────────────────────────────────────
df["revenue"] = df["unit_price"] * df["quantity_sold"]
df["month"] = df["order_date"].dt.month
df["quarter"] = df["order_date"].dt.quarter

# ── 4. DESCRIPTIVE SUMMARY ───────────────────────────────────
print("\nDescriptive stats (clean):")
print(df.describe())
print("\nRevenue by region:")

Example business questions

You choose 3–5. Here are examples — pick from these or invent your own (better questions earn higher scores):

  • Which product category drove the most revenue, and was that consistent across all four regions?
  • Is there a seasonal pattern in sales? Which months are strongest, and by how much above the annual average?
  • How does the Corporate customer segment differ from Consumer in average order size?
  • Are there product categories with a disproportionately high share of returns (negative unit_price rows)?
  • Which region has the highest average order value, and what is driving it — more orders or higher per-order spend?
  • Did revenue grow or decline quarter over quarter? Which quarter had the steepest change?
  • What share of total revenue comes from orders above the 90th percentile? (the "top tier" concentration question)
What "strong" looks like
  • Every cleaning step has a comment explaining the business reason for the decision
  • Every chart has a title, axis labels, and is saved as a PNG (not just displayed)
  • Findings are stated as numbers with context: "Region X drove $Y, which is Z% of total" — never "X seemed to do well"
  • The code runs clean from top to bottom on a freshly cloned repo — no hardcoded paths, no missing imports
  • The README tells a reader how to run your code in 3–5 lines, including which packages to install
  • The presentation is specific and confident: one finding, one chart, one business recommendation
What "weak" looks like
  • Unlabeled charts produced by df.plot() with no formatting — no title, no axis labels
  • Findings stated as opinions: "I think Electronics is popular" or "revenue seems high in the South"
  • Code that crashes or produces errors because cleaning was incomplete
  • A README that just says "run the file" with no context about requirements or execution
  • Three "questions" that are the same question rephrased three ways
  • plt.savefig() called after plt.show(), producing blank PNG files

Chart labeling checklist

Every chart in your capstone must pass this checklist before submission:

  • plt.title("Descriptive Title") — tells the reader what they are looking at
  • plt.xlabel("What this axis measures") — with units if applicable
  • plt.ylabel("What this axis measures") — with units if applicable
  • plt.tight_layout() — prevents clipped labels
  • plt.savefig("charts/chartname.png", dpi=150, bbox_inches="tight") — saved BEFORE show()
  • A one-sentence written interpretation below the chart code (as a comment or markdown cell)

Presentation guide

Your presentation is 5–7 minutes. You do not need slides. You will share your screen and walk through your notebook or script. Structure it as follows:

  1. One sentence context (30 seconds): "I analyzed a retail sales dataset of about 5,000 transactions. After cleaning, I had X rows across Y product categories and four regions."
  2. Your most interesting finding (3–4 minutes): State the question. Show the chart. Read the key number. Explain what it means for the business. What should leadership do differently as a result?
  3. One other finding (1–2 minutes): Brief version of the same structure.
  4. One thing you would do next (30 seconds): If you had more time or more data, what analysis would you run?
